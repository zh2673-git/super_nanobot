# MiroFish 源码解析

## 🎯 第一步：点出核心代码组件

在深入理解代码实现之前，让我们先认识构成这个项目的核心"代码积木块"：

- **[LLMClient]：AI模型调用封装（像"翻译官"，把需求翻译给AI，把AI回复转成结构化数据）**
- **[ProjectManager]：项目持久化管理（像"档案管理员"，负责存储和读取项目信息）**
- **[TaskManager]：异步任务管理（像"进度跟踪器"，记录每个任务的执行状态）**
- **[OntologyGenerator]：本体生成器（调用LLM从文档中提取实体和关系类型）**
- **[GraphBuilderService]：图谱构建器（把文本存到Zep，建立可检索的知识网络）**
- **[SimulationManager]：模拟管理器（管理模拟的整个生命周期）**
- **[OasisProfileGenerator]：Agent人设生成器（为每个实体生成"人设卡片"）**
- **[ReportAgent]：报告代理（带工具集的AI Agent，分析模拟结果）**

**代码组织结构说明**：

MiroFish的代码就像一个"现代化工厂"的不同车间：

1. **API车间**（routes/api_*.py）：负责接收订单（HTTP请求），分发给不同的处理函数
2. **服务中心**（services/）：各个专业车间，每个负责一道工序
   - 本体车间（OntologyGenerator）：理解文档，提取概念
   - 图谱车间（GraphBuilder）：存储知识，建立索引
   - 模拟车间（SimulationManager）：导演"电影"拍摄
   - 报告车间（ReportAgent）：写分析报告
3. **公共设施**（utils/、models/）：通用的工具和模型
   - LLMClient：统一的AI调用窗口
   - ProjectManager：档案管理系统
   - TaskManager：进度跟踪系统

### 1. LLMClient - AI调用封装

**文件位置**：`backend/app/utils/llm_client.py`

**核心功能**：统一调用OpenAI兼容的LLM API

**通俗理解**：就像一个"万能翻译官"，不管你说什么语言（需求），它都能翻译给AI（LLM），也能把AI的回答转成你需要的格式（JSON）。

**关键代码解析**：

```python
class LLMClient:
    """LLM 客户端 - 统一使用 OpenAI SDK"""
    
    def __init__(self):
        self.client = OpenAI(
            api_key=Config.LLM_API_KEY,
            base_url=Config.LLM_BASE_URL
        )
    
    def chat(self, messages, temperature=0.3, max_tokens=4096, **kwargs):
        """
        发送聊天请求
        
        Args:
            messages: 消息列表 [{"role": "user", "content": "..."}]
            temperature: 创造性程度 (0-2)
            max_tokens: 最大返回token数
        """
        response = self.client.chat.completions.create(
            model=Config.LLM_MODEL_NAME,
            messages=messages,
            temperature=temperature,
            max_tokens=max_tokens,
            **kwargs
        )
        return response.choices[0].message.content
```

**使用场景**：
```python
# 场景1：普通对话
llm = LLMClient()
response = llm.chat([{"role": "user", "content": "你好"}])

# 场景2：JSON格式返回
result = llm.chat_json(
    messages=[{"role": "user", "content": "请返回JSON格式的实体列表"}],
    temperature=0.3
)
# 返回: {"entity_types": [...]}
```

```python
    PROJECTS_DIR = "backend/uploads/projects"
    
    @classmethod
    def create_project(cls, name: str) -> Project:
        """创建新项目"""
        project_id = f"proj_{uuid.uuid4().hex[:12]}"
        project = Project(
            project_id=project_id,
            name=name,
            status=ProjectStatus.CREATED,
            created_at=datetime.now().isoformat(),
            updated_at=datetime.now().isoformat()
        )
        # 创建目录并保存
        cls.save_project(project)
        return project
    
    @classmethod
    def save_project(cls, project: Project) -> None:
        """保存项目到JSON文件"""
        meta_path = f"{PROJECTS_DIR}/{project.project_id}/project.json"
        with open(meta_path, 'w', encoding='utf-8') as f:
            json.dump(project.to_dict(), f, ensure_ascii=False, indent=2)
    
    @classmethod
    def get_project(cls, project_id: str) -> Optional[Project]:
        """从文件读取项目"""
        meta_path = f"{PROJECTS_DIR}/{project_id}/project.json"
        if not os.path.exists(meta_path):
            return None
        with open(meta_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        return Project.from_dict(data)
```

### 3. OntologyGenerator - 本体生成

**文件位置**：`backend/app/services/ontology_generator.py`

**核心功能**：分析文档内容，生成实体类型和关系类型定义

**通俗理解**：就像一个"图书分类员"，读了一本书后，总结出这本书涉及哪些"人物"，这些人物之间是什么"关系"。

**核心Prompt设计**：

```python
# 系统提示词的核心要求
ONTOLOGY_SYSTEM_PROMPT = """
你是一个知识图谱本体设计专家。
你的任务是分析文本内容，设计适合**社交媒体舆论模拟**的实体类型和关系类型。

重要规则：
1. 实体必须是可以在社交媒体上发声的主体（个人、组织、媒体等）
2. 不能是抽象概念（舆论、情绪、趋势）
3. 必须正好输出10个实体类型
4. 最后2个必须是兜底类型：Person 和 Organization
"""
```

**生成流程**：

```python
class OntologyGenerator:
    def generate(
        self,
        document_texts: List[str],
        simulation_requirement: str,
        additional_context: str = None
    ) -> Dict[str, Any]:
        """
        生成本体定义
        
        输入：文档文本 + 模拟需求
        输出：实体类型列表 + 关系类型列表 + 分析摘要
        """
        # 1. 合并文本（最多5万字）
        combined_text = "\n\n---\n\n".join(document_texts)
        
        # 2. 构建用户消息
        user_message = f"""
        ## 模拟需求
        {simulation_requirement}
        
        ## 文档内容
        {combined_text}
        """
        
        # 3. 调用LLM
        result = self.llm_client.chat_json(messages=[
            {"role": "system", "content": ONTOLOGY_SYSTEM_PROMPT},
            {"role": "user", "content": user_message}
        ])
        
        # 4. 验证和后处理
        return self._validate_and_process(result)
```

### 4. GraphBuilderService - 图谱构建

**文件位置**：`backend/app/services/graph_builder.py`

**核心功能**：将文本存到Zep图谱服务，建立可检索的知识网络

**通俗理解**：就像一个"图书馆员"，把书切分成章节，编上索引，放到书架上，方便以后快速查找。

**核心流程**：

```python
class GraphBuilderService:
    def __init__(self, api_key: str):
        self.client = ZepClient(api_key=api_key)
    
    def create_graph(self, name: str) -> str:
        """创建图谱"""
        graph = self.client.graph.create(name=name)
        return graph.uuid
    
    def set_ontology(self, graph_id: str, ontology: Dict):
        """设置本体定义"""
        entity_types = ontology.get("entity_types", [])
        edge_types = ontology.get("edge_types", [])
        
        # 转换为Zep格式
        self.client.ontology.entities.create(graph_id, entities=entity_types)
        self.client.ontology.edges.create(graph_id, edges=edge_types)
    
    def add_text_batches(self, graph_id, chunks, batch_size=3):
        """批量添加文本"""
        episode_uuids = []
        
        for i in range(0, len(chunks), batch_size):
            batch = chunks[i:i+batch_size]
            
            # 添加文档
            response = self.client.graph.add_documents(
                graph_id=graph_id,
                documents=[{"content": chunk} for chunk in batch]
            )
            
            episode_uuids.extend([r.uuid for r in response])
        
        return episode_uuids
    
    def _wait_for_episodes(self, episode_uuids, progress_callback):
        """等待Zep处理完成"""
        while True:
            all_processed = True
            for uuid in episode_uuids:
                episode = self.client.document.get(uuid)
                if not episode.processed:
                    all_processed = False
                    break
            
            if all_processed:
                break
            time.sleep(2)  # 每2秒检查一次
```

**准备流程**（prepare_simulation）：

```python
def prepare_simulation(
    self,
    simulation_id: str,
    simulation_requirement: str,
    document_text: str,
    defined_entity_types: List[str] = None,
    progress_callback = None
) -> Simulation:
    """
    准备模拟环境
    """
    # 阶段1: 读取图谱实体 (0-20%)
    if progress_callback:
        progress_callback("reading", 0, "读取图谱实体...")
    
    reader = ZepEntityReader()
    entities = reader.filter_defined_entities(
        graph_id=self.graph_id,
        defined_entity_types=defined_entity_types,
        enrich_with_edges=True
    )
    
    # 阶段2: 生成Agent人设 (20-70%)
    if progress_callback:
        progress_callback("generating_profiles", 20, "生成Agent人设...")
    
    profile_gen = OasisProfileGenerator()
    profiles = profile_gen.generate_profiles(
        entities=entities,
        simulation_requirement=simulation_requirement,
        document_text=document_text
    )
    
    # 阶段3: 生成模拟配置 (70-90%)
    if progress_callback:
        progress_callback("generating_config", 70, "生成模拟配置...")
    
    config_gen = SimulationConfigGenerator()
    config = config_gen.generate_config(
        simulation_requirement=simulation_requirement,
        entities=entities
    )
    
    # 阶段4: 保存配置文件 (90-100%)
    self._save_config(simulation_id, profiles, config)
    
    # 更新状态
    state.status = SimulationStatus.READY
    self._save_simulation_state(state)
    
    return state
```

### 6. OasisProfileGenerator - Agent人设生成

**文件位置**：`backend/app/services/oasis_profile_generator.py`

**核心功能**：为每个实体生成"人设卡片"，让AI可以扮演这个角色

**通俗理解**：就像一个"选角导演"，根据剧本中的角色设定，为每个角色生成详细的"人物小传"（包括性格、说话风格、立场等）。

**生成逻辑**：

```python
class OasisProfileGenerator:
    def generate_profile_for_entity(
        self,
        entity: Dict,
        simulation_requirement: str,
        document_context: str,
        platform: str = "reddit"
    ) -> Dict:
        """
        为单个实体生成人设
        """
        # 构建Prompt
        prompt = f"""
        你是一个角色设计师。请根据以下信息，为这个实体生成一个完整的人设。
        
        实体信息：{entity}
        模拟需求：{simulation_requirement}
        文档背景：{document_context[:5000]}
        
        请生成以下信息：
        1. 姓名（保持或稍作修改）
        2. 用户名（符合{platform}风格）
        3. 头像描述
        4. 性格特点
        5. 立场（支持/反对/中立）
        6. 发言风格
        7. 背景故事
        """
        
        # 调用LLM
        result = self.llm_client.chat_json([
            {"role": "system", "content": PROFILE_SYSTEM_PROMPT},
            {"role": "user", "content": prompt}
        ])
        
        # 转换为OASIS格式
        return self._convert_to_oasis_format(result, platform)
    
    def generate_profiles(self, entities, ...):
        """批量生成人设"""
        profiles = []
        for i, entity in enumerate(entities):
            profile = self.generate_profile_for_entity(entity, ...)
            profiles.append(profile)
            
            # 进度回调
            if progress_callback:
                progress_callback(
                    stage="generating_profiles",
                    progress=int((i + 1) / len(entities) * 50),
                    current=i+1,
                    total=len(entities)
                )
        
        return profiles
```

```python
        """
        # 构建系统提示
        system_prompt = f"""
        你是一个专业的舆情分析师。
        你可以调用各种工具来分析模拟结果，回答用户的问题。
        
        模拟ID: {self.simulation_id}
        
        请遵循以下步骤：
        1. 理解用户问题
        2. 调用相关工具获取数据
        3. 分析数据
        4. 给出专业的分析报告
        """
        
        # 调用LLM（带工具）
        response = self.llm_client.chat_with_tools(
            messages=[
                {"role": "system", "content": system_prompt},
                {"role": "user", "content": requirement}
            ],
            tools=self.tools
        )
        
        # 处理工具调用
        # ... (处理search_posts, analyze_sentiment等工具调用)
        
        # 返回最终报告
        return self._format_report(response)
    
    def interview_agent(self, username: str, question: str) -> str:
        """
        与模拟世界中的Agent对话
        """
        # 获取Agent的人设和记忆
        agent_info = self._get_agent_info(username)
        
        # 构建对话Prompt
        prompt = f"""
        结合你的人设和所有记忆，直接回答我的问题。
        
        人设: {agent_info['persona']}
        记忆: {agent_info['memory']}
        
        问题: {question}
        
        注意：不要调用工具，直接用文本回答。
        """
        
        response = self.llm_client.chat([
            {"role": "user", "content": prompt}
        ])
        
        return response
```

### 8. TaskManager - 异步任务管理

**文件位置**：`backend/app/models/task.py`

**核心功能**：管理异步任务的执行状态，供前端轮询进度

**通俗理解**：就像一个"进度跟踪器"，当你点了一个"耗时操作"（如构建图谱），它给你一个"取餐号"（task_id），你可以随时查询"菜做得怎么样了"（进度）。

**任务状态**：

```python
class TaskStatus(str, Enum):
    PENDING = "pending"       # 等待执行
    PROCESSING = "processing" # 执行中
    COMPLETED = "completed"  # 完成
    FAILED = "failed"         # 失败

@dataclass
class Task:
    task_id: str
    task_type: str            # 任务类型
    status: TaskStatus
    progress: int             # 0-100
    message: str              # 进度描述
    result: Any = None        # 任务结果
    error: str = None         # 错误信息
    
    created_at: str
    updated_at: str
```

**使用流程**：

```python
# 1. 创建任务
task_manager = TaskManager()
task_id = task_manager.create_task("构建图谱")

# 2. 立即返回给前端
return jsonify({
    "task_id": task_id,
    "message": "任务已启动"
})

# 3. 后台执行
def background_task():
    task_manager.update_task(task_id, status=TaskStatus.PROCESSING, progress=10)
    # ... 执行中 ...
    task_manager.update_task(task_id, progress=50, message="正在处理...")
    # ... 
    task_manager.complete_task(task_id, result={"graph_id": "..."})

# 4. 前端轮询
# GET /api/task/{task_id}
# 返回: {"status": "processing", "progress": 50, "message": "正在处理..."}
```
