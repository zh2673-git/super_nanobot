# MiroFish 项目概述

## 🎯 第一步：点出核心概念

在深入了解这个项目之前，让我们先认识它的核心"积木块"：

- **[MiroFish]：一款基于多智能体技术的AI预测引擎（就像一个"数字世界模拟器"，可以把现实世界的种子信息投放到虚拟世界中，观察未来可能发生的故事）**
- **[群体智能]：多个具备独立人格的智能体进行自由交互（像是一场大型"角色扮演游戏"，每个角色都有自己的性格和记忆）**
- **[OASIS框架]：驱动仿真引擎的核心框架（就像汽车的发动机，提供模拟社交媒体交互的能力）**
- **[GraphRAG]：知识图谱检索增强技术（像是一个"超级图书馆管理员"，能快速找到相关的背景知识）**
- **[Zep Cloud]：长期记忆服务（像是一个"记忆宫殿"，让智能体能记住长期积累的信息）**

**联系说明**：MiroFish就像一个"数字平行世界"建造者。你把现实中的种子材料（新闻、小说、报告）放进去，它会：
1. 先"消化"这些材料，构建知识图谱（像图书管理员整理书籍）
2. 然后生成无数个有"人设"的智能体（像导演创造出众多角色）
3. 让这些智能体在模拟的社交媒体上自由互动（像观察一个微型社会的演化）
4. 最后生成预测报告，并让你能和任何虚拟角色对话（像获得了"上帝视角"和"读心术"）

## 📖 第三步：详细解释

### 1. MiroFish 是什么？

MiroFish的中文名是"鳐鱼"，寓意它在信息海洋中灵活游动、捕捉趋势。它是一个**群体智能预测引擎**，核心思想是：

> "让未来在数字沙盘中预演，助决策在百战模拟后胜出"

**通俗理解**：就像电影《盗梦空间》中的梦境推演，MiroFish把你提供的信息（种子）变成一个虚拟世界，然后让无数个AI角色在里面生活、互动、争论，最后观察他们的发展轨迹，从而预测现实世界可能发生什么。

### 2. 核心功能

MiroFish主要解决两类问题：

**宏观层面（决策者）**：
- 政策推演：新政策发布后，网民会如何反应？
- 舆情预测：突发新闻会如何发酵、发酵到什么程度？
- 公关模拟：危机事件发生后，不同群体的态度会如何演变？

**微观层面（个人用户）**：
- 故事推演：《红楼梦》后40回可能是什么结局？
- 创意探索：如果某种设定出现在现实世界，会发生什么？
- 角色互动：和一个虚构角色深入对话会是什么体验？

### 3. 技术栈

MiroFish采用了现代化的技术架构：

**前端**：
- Vue.js 3（构建用户界面）
- Node.js 18+（运行前端服务）

**后端**：
- Python 3.11-3.12（核心逻辑）
- Flask（Web框架，提供API接口）
- OASIS框架（CAMEL-AI团队开发的社会模拟引擎）

**AI能力**：
- OpenAI SDK兼容接口（支持任意LLM）
- 推荐使用阿里云百炼的qwen-plus模型
- GraphRAG（知识图谱检索增强）
- Zep Cloud（长期记忆服务）

**部署**：
- Docker（容器化部署）
- 同时支持源码部署

### 4. 五大工作流程

MiroFish的预测过程分为五个步骤，每一步都像工厂的一条生产线：

**第一步：图谱构建**
- 输入：种子材料（数据分析报告、小说故事等）
- 处理：实体关系抽取、知识图谱构建、GraphRAG索引
- 产出：结构化的知识网络

**第二步：环境搭建**
- 输入：知识图谱
- 处理：提取关键人物、生成每个人设、配置模拟参数
- 产出：可运行的虚拟环境

**第三步：开始模拟**
- 输入：模拟需求
- 处理：OASIS引擎驱动Twitter和Reddit双平台并行模拟
- 产出：智能体的互动数据

**第四步：报告生成**
- 输入：模拟数据
- 处理：ReportAgent（报告代理）深度分析
- 产出：预测报告

**第五步：深度互动**
- 输入：用户查询
- 处理：与任意智能体或ReportAgent对话
- 产出：实时交互反馈

### 6. 快速开始

如果你想本地运行MiroFish：

```bash
# 1. 克隆项目（已在本地）
# 2. 配置环境变量
cp .env.example .env
# 编辑.env，填入LLM_API_KEY和ZEP_API_KEY

# 3. 一键安装
npm run setup:all

# 4. 启动服务
npm run dev
# 前端: http://localhost:3000
# 后端: http://localhost:5001
```
