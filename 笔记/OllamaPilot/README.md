# OllamaPilot 项目笔记

> 基于 LangChain 1.0+ 的本地智能助手框架，让 4B 小模型也能稳定执行复杂任务

---

## 🎯 第一步：认识三大核心组件

OllamaPilot 就像一座设计精良的工厂，由三个核心"车间"协同工作：

- **Agent（总调度）**：整个系统的大脑和指挥中心，像工厂的总调度室（负责接收用户指令、协调各个车间工作、返回最终结果）
- **Skill（技能车间）**：独立的功能模块，即插即用，像工厂的专业车间（每个车间负责特定类型的任务，如天气查询、代码执行、文件操作等）
- **Middleware（中间件流水线）**：处理环节的流水线，像工厂的传送带和质检站（负责工具重试、日志记录、Skill 选择等）

**为什么这样设计？**

想象你要建一个工厂：
- 如果所有功能都堆在一起，就像一个车间又要造零件、又要包装、又要质检，乱成一团
- OllamaPilot 的设计就像把工厂分成独立的车间，每个车间专注做好自己的事
- Agent 像总调度室，知道什么时候该叫哪个车间
- Skill 像专业车间，来了任务就专注做好
- Middleware 像传送带和质检站，确保每个环节都正常运转

**协作流程说明**：
1. **用户提问**："明天苏州天气怎么样？"
2. **Agent 接收**：总调度室收到请求，开始处理
3. **Skill 选择**：SkillSelector 发现"天气"关键词，激活 Weather Skill
4. **工具执行**：Weather Skill 调用 web_search 搜索天气信息
5. **日志记录**：ToolLogging 记录"搜索天气"的操作
6. **失败处理**：如果搜索失败，ToolRetry 自动重试
7. **结果返回**：整理天气信息，返回给用户

## 📚 第三步：深入理解每个组件

### 一、Agent（总调度）

**专业定义**：Agent 是 OllamaPilot 的核心调度器，基于 LangChain 的 `create_agent` 工厂函数创建，负责管理整个对话流程、协调各个组件。

**通俗解释与类比**：
想象一个公司的前台接待员：
- 客户来了，前台要问清楚客户的需求
- 根据需求，前台要判断应该叫哪个部门来处理
- 部门处理完后，前台要整理结果并回复客户
- 如果部门处理失败，前台要安排重试

Agent 就是这个"前台接待员+调度中心"，它知道：
- 如何理解用户的问题
- 问题应该由哪个 Skill 处理
- 处理过程中需要哪些工具
- 如果出错应该如何恢复

**在项目中的具体体现**：
Agent 的核心工作流程：
1. **初始化阶段**：
   - 加载所有 Skill（像前台拿到各部门联系表）
   - 加载所有工具（像前台了解各部门能做什么）
   - 配置中间件（像前台设定工作流程）

2. **对话处理阶段**：
   - 接收用户输入
   - 查找匹配的 Skill（比如检测到"天气"关键词，激活 Weather Skill）
   - 获取 Skill 的系统提示词（给模型明确的指令）
   - 调用模型生成回复
   - 管理对话记忆（记住之前聊过什么）

3. **错误处理阶段**：
   - 工具调用失败时自动重试
   - 记录详细的执行日志
   - 防止无限循环（限制最多调用 50 次工具）

### 二、Skill（技能车间）

**专业定义**：Skill 是独立的功能模块，遵循"USB 即插即用"设计理念，可以独立开发、测试、部署。每个 Skill 包含：名称、描述、触发词、工具列表、系统提示词。

**通俗解释与类比**：
想象你有一套乐高积木：
- 每块积木都是独立的，但可以拼在一起
- 你可以随时加一块新积木，不影响其他积木
- 每块积木有自己的功能（这个是轮子，那个是窗户）

Skill 就像这些乐高积木：
- 每个 Skill 是独立的功能模块
- 可以随时添加新 Skill，不影响现有的 Skill
- Skill 之间互不干扰，各司其职

**Skill 的三种创建方式**：

1. **SKILL.md（0 行代码）**：
   ```yaml
   ---
   name: weather
   description: 查询天气信息
   triggers: [天气, 温度, 下雨]
   tools: [web_search, web_fetch]
   ---
   
   你是天气查询专家...
   ```
   - 就像写配置文件一样简单
   - 无需写任何 Python 代码
   - 适合简单、明确的功能

2. **Python Skill（20 行代码）**：
   ```python
   class WeatherSkill(Skill):
       name = "weather"
       description = "查询天气信息"
       triggers = ["天气", "温度"]
       
       def get_tools(self):
           return [get_weather_tool]
       
       def get_system_prompt(self):
           return "你是天气专家..."
   ```
   - 需要写简单的 Python 代码
   - 可以添加自定义逻辑
   - 适合中等复杂度的功能

3. **MCP + 自定义（50 行代码）**：
   - 接入外部服务（如 MCP 服务器）
   - 最灵活但最复杂
   - 适合复杂集成场景

**在项目中的具体体现**：
项目的 `skills/weather/SKILL.md` 定义了天气查询 Skill：
- **触发词**：天气、温度、下雨
- **工具**：web_search、web_fetch
- **提示词**：告诉模型如何查询和整理天气信息

### 三、Middleware（中间件流水线）

**专业定义**：Middleware 是 LangChain 的 AgentMiddleware 机制，在 Agent 执行流程中插入自定义处理逻辑。OllamaPilot 使用中间件实现：Skill 选择、工具调用日志、自动重试、调用限制。

**通俗解释与类比**：
想象一个快递分拣线：
- 包裹先经过"扫码台"，识别目的地
- 然后经过"记录台"，记录包裹信息
- 接着经过"称重台"，检查是否超重
- 最后经过"安检台"，检查是否违禁

每个"台"都是一个中间件：
- 可以随时加新的"台"（添加新功能）
- 可以调整"台"的顺序（改变执行顺序）
- 每个"台"独立工作，互不干扰

**在项目中的具体体现**：
OllamaPilot 的中间件执行顺序：

1. **SkillSelectorMiddleware**（Skill 选择器）：
   - 像快递的"扫码台"，识别包裹应该去哪
   - 检查用户输入的关键词，选择合适的 Skill
   - 例：检测到"天气"，选择 Weather Skill

2. **ToolLoggingMiddleware**（工具日志）：
   - 像快递的"记录台"，记录每个包裹的去向
   - 记录工具调用的名称、参数、结果
   - 方便调试和追踪问题

3. **ToolRetryMiddleware**（工具重试）：
   - 像快递的"失败处理台"，包裹送不掉就重试
   - 工具调用失败时自动重试（最多 2 次）
   - 提高系统的稳定性

4. **ToolCallLimitMiddleware**（调用限制）：
   - 像快递的"时间限制台"，防止无限等待
   - 限制工具调用次数（最多 50 次）
   - 防止模型陷入死循环

## 🗂️ 第四步：项目目录结构

```
OllamaPilot/
├── ollamapilot/              # 核心代码包
│   ├── __init__.py           # 包入口，导出公共接口
│   ├── agent.py              # Agent 核心（总调度）
│   ├── skill_middleware.py   # 中间件系统（流水线）
│   ├── models.py             # 模型管理（连接 Ollama）
│   ├── skills/               # Skill 系统
│   │   ├── base.py           # Skill 基类定义
│   │   ├── loader.py         # SKILL.md 加载器
│   │   ├── registry.py       # Skill 注册中心
│   │   └── default_skill.py  # 默认 Skill
│   └── tools/                # 工具系统
│       ├── builtin.py        # 9 个内置工具
│       ├── mcp_tools.py      # MCP 工具支持
│       └── custom/           # 自定义工具目录
├── skills/                   # 用户 Skill 目录
│   └── weather/
│       └── SKILL.md          # 天气查询 Skill
├── tests/                    # 测试套件
│   └── test_agent.py
├── main.py                   # CLI 入口
├── pyproject.toml            # 项目配置
└── README.md                 # 项目说明
```

**关键文件说明**：
- **agent.py**：创建 Agent，配置中间件，管理对话流程
- **skill_middleware.py**：实现 Skill 选择、工具日志、重试逻辑
- **models.py**：初始化 Ollama 模型，查询可用模型列表
- **skills/base.py**：定义 Skill 的标准接口
- **tools/builtin.py**：Web 搜索、文件操作等 9 个工具

## 🚀 快速开始

### 最简使用（3 行代码）

```python
from ollamapilot import init_ollama_model, create_agent

model = init_ollama_model("qwen3.5:4b")
agent = create_agent(model, skills_dir="skills")
response = agent.invoke("明天苏州天气怎么样？")
```

### 运行交互式对话

```bash
python main.py
```

### 创建自定义 Skill

在 `skills/my_skill/SKILL.md` 中：

```yaml
---
name: my_skill
description: 我的自定义技能
triggers: [关键词1, 关键词2]
tools: [web_search, python_exec]
---

你是一个专业的助手，帮助用户解决相关问题...
```
