# OllamaPilot 工具系统机制详解

> **创建时间**: 2026-03-08  
> **项目**: OllamaPilot - 基于 LangChain 1.0+ 的本地智能助手框架  
> **核心特性**: 让 4B 小模型也能稳定执行复杂任务

---

## 🧩 第一步：点出工具系统的组件概念

OllamaPilot 的工具系统由以下几个核心组件构成,每个组件都有明确的职责:

- **工具函数（Tool Functions）**: 执行具体操作的函数,如读文件、执行代码等（就像工具箱里的螺丝刀、扳手,每个工具完成特定任务）
- **工具装饰器（@tool Decorator）**: 将普通函数转换为 LangChain 工具的装饰器（就像给工具贴上标签和说明书,让AI知道如何使用）
- **工具注册机制（Tool Registry）**: 管理和索引所有可用工具的系统（就像工具仓库的管理员,知道仓库里有哪些工具、放在哪里）
- **Skill 中间件（Skill Middleware）**: 根据用户需求选择合适工具的智能调度系统（就像工厂的车间主任,根据订单安排合适的工人和工具）
- **安全防护机制（Safety Mechanisms）**: 防止危险操作的安全检查系统（就像工具的安全锁定装置,防止误操作造成损害）

---

## 🔗 第二步：用图构建组件概念之间的联系

```
[用户输入："帮我分析一下data.txt文件"]
    |
    v
[AI 模型 - 大脑]
    |-> 理解用户意图
    |-> 决定需要使用哪些工具
    |
    v
[Skill 选择器中间件]
    |-> 分析关键词："分析"、".txt文件"
    |-> 匹配 Skill 触发词
    |-> 选择合适的 Skill（如"文件分析 Skill"）
    |-> 注入该 Skill 的系统提示词
    |
    v
[工具调用日志中间件]
    |-> 记录工具调用开始
    |-> 显示: 🔧 执行工具: read_file
    |
    v
[工具重试中间件]
    |-> 调用工具函数
    |-> 如果失败,自动重试（最多2次）
    |
    v
[工具限流中间件]
    |-> 检查工具调用次数（最多50次）
    |-> 防止无限循环
    |
    v
[安全检查机制]
    |-> 检查工具参数是否安全
    |-> 检查是否包含危险操作
    |-> 通过 -> 执行工具
    |-> 拒绝 -> 返回错误信息
    |
    v
[工具函数执行]
    |-> read_file 函数被调用
    |-> 参数: file_path="data.txt"
    |-> 执行文件读取操作
    |-> 返回文件内容
    |
    v
[工具调用日志中间件]
    |-> 记录工具调用结果
    |-> 显示: ✅ 结果: 📄 文件: data.txt...
    |
    v
[AI 模型 - 大脑]
    |-> 接收工具返回的结果
    |-> 理解文件内容
    |-> 生成分析报告
    |
    v
[返回结果给用户]
```

**联系说明**：当用户输入一个请求时,AI模型（大脑）先理解意图,决定需要使用哪些工具；Skill 选择器（车间主任）根据关键词匹配合适的 Skill；工具调用日志中间件（记录员）记录每一步操作；工具重试中间件（保镖）确保操作成功,失败了会重试；工具限流中间件（安保）防止过度调用；安全检查机制（安全锁）拦截危险操作；最后工具函数（具体工具）执行实际操作,并将结果一步步返回给用户。

## 📖 第三步：详细解释每个组件

### 一、工具函数（Tool Functions）

#### 专业定义
工具函数是封装了具体操作逻辑的 Python 函数,通过 `@tool` 装饰器转换为 LangChain 工具对象。工具函数定义了操作的输入参数、执行逻辑和输出结果,是 Agent 与外部世界交互的主要接口。

#### 通俗解释与类比
想象你有一套齐全的工具箱：
- **螺丝刀**（read_file）：用来拧螺丝（读取文件）
- **电钻**（shell_exec）：用来钻孔（执行命令）
- **计算器**（python_exec）：用来算数（执行代码）
- **浏览器**（web_search）：用来搜索信息（网络搜索）

每个工具都有自己的使用方法（参数）和用途（功能说明）。当你需要做某件事时,只需要拿出对应的工具。

### 二、工具装饰器（@tool Decorator）

#### 专业定义
`@tool` 是 LangChain 提供的装饰器,用于将普通 Python 函数转换为 LangChain 工具对象。它会自动提取函数的类型注解和文档字符串,生成工具的 JSON Schema,供模型理解和调用。

#### 通俗解释与类比
想象你发明了一个新工具（Python 函数）,但别人不知道怎么用。你需要在工具上贴一个标签:
1. **工具名称**：就是这个函数的名字
2. **使用说明书**：就是函数的文档字符串（docstring）
3. **参数要求**：就是函数的类型注解（type hints）

`@tool` 装饰器就像一个"标签打印机",自动帮你生成这些标签,让 AI 能够读懂工具的使用方法。

#### 在项目中的具体体现

**装饰器的工作原理**：

```python
from langchain_core.tools import tool

@tool  # 这个装饰器做了什么？
def read_file(file_path: str, limit: int = 1000) -> str:
    """
    读取文件内容
    
    Args:
        file_path: 文件路径（相对或绝对）
        limit: 最大读取行数,默认1000行
        
    Returns:
        文件内容
    """
    # ... 具体实现 ...

# 装饰器自动提取的信息：
# {
#   "name": "read_file",
#   "description": "读取文件内容",
#   "parameters": {
#     "type": "object",
#     "properties": {
#       "file_path": {
#         "type": "string",
#         "description": "文件路径（相对或绝对）"
#       },
#       "limit": {
#         "type": "integer",
#         "description": "最大读取行数,默认1000行",
#         "default": 1000
#       }
#     },
#     "required": ["file_path"]
#   }
# }
```

**关键点**：
1. **自动提取函数名**：`read_file` 成为工具名称
2. **自动解析文档**：文档字符串的第一句成为工具描述
3. **自动提取参数类型**：`file_path: str` 表示参数类型为字符串
4. **自动识别必需参数**：没有默认值的参数标记为 required
5. **自动识别可选参数**：有默认值的参数标记为 optional

**为什么这样设计**：
- AI 模型需要知道工具的"使用说明书"才能正确调用
- 通过自动提取,开发者无需手动编写 JSON Schema
- 保持代码简洁,一处定义,多处使用

### 三、工具注册机制（Tool Registry）

#### 专业定义
工具注册机制是一个管理系统,负责收集、索引和提供所有可用工具。在 OllamaPilot 中,工具注册主要体现在 Agent 初始化时收集内置工具,并绑定到模型上。

#### 通俗解释与类比
想象一个工具仓库管理员:
1. **工具入库**：管理员接收新工具,登记入库（收集工具函数）
2. **工具索引**：给每个工具分配编号和存放位置（建立工具名称到对象的映射）
3. **工具查找**：当有人需要工具时,管理员快速找到并递给他（提供工具实例）

在 OllamaPilot 中,`_get_builtin_tools()` 方法就像"工具盘点清单",列出了所有可用的工具。

#### 在项目中的具体体现

**工具收集流程**：

### 四、Skill 中间件系统（Skill Middleware System）

#### 专业定义
Skill 中间件系统是基于 LangChain AgentMiddleware 机制的智能调度层。它由多个中间件组成,按照顺序执行,负责 Skill 选择、工具调用日志、工具重试、工具限流等功能。

#### 通俗解释与类比
想象一个现代化的智能工厂:

1. **Skill 选择器中间件（SkillSelectorMiddleware）** = **车间主任**
   - 接到订单（用户请求）后,判断应该由哪个车间（哪个 Skill）来处理
   - 根据订单关键词匹配车间专长（根据触发词匹配 Skill）
   - 给车间分配任务,提供指导书（注入系统提示词）

2. **工具调用日志中间件（ToolLoggingMiddleware）** = **记录员**
   - 记录每个工人（工具）开始工作的时间和任务参数
   - 记录工作完成后的结果
   - 方便后续追溯和调试

3. **工具重试中间件（ToolRetryMiddleware）** = **质检员**
   - 检查工人的工作成果
   - 如果发现失败,要求工人重做（最多2次）
   - 确保任务最终成功

4. **工具限流中间件（ToolCallLimitMiddleware）** = **安保人员**
   - 监控工具调用次数
   - 防止某个工人在循环中无限工作
   - 设置上限（默认50次）,超过则强制停止

#### 在项目中的具体体现

**中间件构建流程**：

```python
# 文件: ollamapilot/agent.py

def _build_middleware(self, max_tool_calls: int) -> List[Any]:
    """
    构建中间件列表
    
    中间件执行顺序（从前到后）:
    1. SkillSelectorMiddleware - Skill 选择
    2. ToolLoggingMiddleware - 工具调用日志
    3. ToolRetryMiddleware - 工具重试
    4. ToolCallLimitMiddleware - 工具调用限制
    """
    middleware = []
    
    # 1. Skill 选择器（核心）
    selector = SkillSelectorMiddleware(
        self.skill_registry,
        default_skill_name="default",
        verbose=self.verbose
    )
    middleware.append(selector)
    
    # 2. 工具调用日志
    if self.verbose:
        logging_mw = ToolLoggingMiddleware(verbose=True)
        middleware.append(logging_mw)
    
    # 3. 工具重试
    retry_mw = ToolRetryMiddleware(max_retries=2)
    middleware.append(retry_mw)
    
    # 4. 工具调用限制
    limit_mw = ToolCallLimitMiddleware(run_limit=max_tool_calls)
    middleware.append(limit_mw)
    
    return middleware
```

**Skill 选择器的核心逻辑**：

```python
# 文件: ollamapilot/skill_middleware.py

class SkillSelectorMiddleware(AgentMiddleware):
    """Skill 选择器中间件"""
    
    def before_model(self, state: Any, runtime: Any) -> Optional[Dict[str, Any]]:
        """
        选择合适的 Skill 并注入系统提示词
        
        执行时机：在模型调用前
        执行逻辑：
        1. 获取用户最新消息
        2. 根据关键词匹配 Skill
        3. 注入 Skill 的系统提示词
        """
        # 获取最后一条用户消息
        messages = state.get("messages", [])
        last_message = messages[-1]
        query = str(last_message.content)
        
        # 查找匹配的 Skill
        skill = self._select_skill(query)
        
        if skill:
            # 注入系统提示词
            system_prompt = skill.get_system_prompt()
            if system_prompt:
                return {
                    "messages": [SystemMessage(content=system_prompt)] + messages
                }
        
        return None
    
    def _select_skill(self, query: str) -> Optional[Skill]:
        """根据查询选择合适的 Skill"""
        # 1. 查找匹配的特定 Skill
        matches = self.registry.find_skill_by_trigger(query)
        
        if matches:
            return self.registry.get_skill(matches[0])
        
        # 2. 返回默认 Skill
        return self.registry.get_skill("default")
```

```python
    def wrap_tool_call(self, request: Any, handler: Any) -> Any:
        """
        包装工具调用,添加日志
        
        执行时机：每次工具调用时
        执行逻辑：
        1. 记录工具名称和参数
        2. 执行工具调用
        3. 记录返回结果
        """
        if self.verbose:
            tool_name = request.tool_call.get("name", "unknown")
            tool_args = request.tool_call.get("args", {})
            print(f"🔧 执行工具: {tool_name}({tool_args})")
        
        result = handler(request)  # 执行工具
        
        if self.verbose:
            result_preview = str(result)[:200]
            print(f"   ✅ 结果: {result_preview}")
        
        return result
```

**中间件执行顺序图**：

```

    ↓
[1. SkillSelectorMiddleware]
    ↓ 选择合适的 Skill,注入系统提示词
[2. ToolLoggingMiddleware]
    ↓ 记录工具调用开始
[3. ToolRetryMiddleware]
    ↓ 如果失败,自动重试
[4. ToolCallLimitMiddleware]
    ↓ 检查是否超过限制
[AI 模型决策]
    ↓ 决定调用哪个工具
[工具执行]
    ↓
[返回结果]
    ↓
[工具调用日志中间件]
    ↓ 记录工具调用结果
[返回给用户]
```

### 五、安全防护机制（Safety Mechanisms）

#### 专业定义
安全防护机制是一系列检查和限制措施,防止工具执行危险操作。主要包括命令黑名单、代码安全检查、文件操作限制等。

#### 通俗解释与类比
想象工具箱上的安全锁定装置:
1. **危险命令黑名单**：就像工具箱上贴着"禁止使用"的红标签,列出了一堆危险的命令（如 `rm -rf /`）
2. **代码安全检查**：就像一个代码审查员,在你运行代码前检查是否有危险操作（如 `os.system`, `subprocess`）
3. **超时限制**：就像定时器,如果操作运行太久,自动停止,防止卡死

#### 在项目中的具体体现

**1. Shell 命令安全检查**：

```python
# 文件: ollamapilot/tools/builtin.py

# 危险命令黑名单
DANGEROUS_COMMANDS = [
    'rm -rf /',           # 删除根目录
    'rm -rf /*',          # 删除根目录下所有文件
    'rm -rf ~',           # 删除用户主目录
    'mkfs',               # 格式化磁盘
    'dd if=',             # 磁盘复制（可能覆盖系统）
    '>:',                 # 端口重定向（可能被滥用）
    'format',             # 格式化
    'del /f /s /q',       # Windows 强制删除
    'rd /s /q',           # Windows 递归删除
]

def _is_dangerous_command(command: str) -> bool:
    """检查命令是否危险"""
    cmd_lower = command.lower().strip()
    for dangerous in DANGEROUS_COMMANDS:
        if dangerous in cmd_lower:
            return True
    return False

@tool
def shell_exec(command: str, timeout: int = 30) -> str:
    """执行 Shell 命令"""
    # 安全校检
    if _is_dangerous_command(command):
        return f"🚫 安全拦截: 检测到危险命令"
    
    # 执行命令
    result = subprocess.run(...)
```

**生活化类比**：就像一个门卫,检查每个要进入的人。如果发现有人拿着"炸药"（危险命令）,就立即阻止他进入。

**2. Python 代码安全检查**：

```python
# 文件: ollamapilot/tools/builtin.py

# 危险操作黑名单
DANGEROUS_MODULES = [
    'os.system',      # 执行系统命令
    'os.popen',       # 打开管道
    'subprocess',     # 子进程管理
    'sys.exit',       # 退出程序
    '__import__',     # 动态导入
    'eval',           # 动态执行表达式
    'exec',           # 动态执行代码
    'compile',        # 编译代码
]

def _check_code_safety(code: str) -> tuple[bool, str]:
    """检查代码安全性"""
    code_lower = code.lower()
    
    # 检查危险模块
    for dangerous in DANGEROUS_MODULES:
        if dangerous in code_lower:
            return False, f"🚫 安全拦截: 检测到危险操作 '{dangerous}'"
    
    # 检查危险文件操作
    dangerous_patterns = [
        'rm -rf', 'del /', 'format(', 'mkfs',
        'shutil.rmtree', 'os.rmdir', 'os.remove',
    ]
    for pattern in dangerous_patterns:
        if pattern in code_lower:
            return False, f"🚫 安全拦截: 检测到危险文件操作 '{pattern}'"
    
    return True, ""

@tool
def python_exec(code: str, timeout: int = 30) -> str:
    """执行 Python 代码片段"""
    # 安全校验
    is_safe, error_msg = _check_code_safety(code)
    if not is_safe:
        return error_msg
    
    # 执行代码
    exec(code, env)
```

**生活化类比**：就像一个沙箱实验室,在这个实验室里可以安全地做实验,但不能触碰危险品（危险模块）,也不能破坏实验室设施（危险文件操作）。

**3. 超时限制**：

```python
@tool
def shell_exec(command: str, timeout: int = 30) -> str:
    """执行 Shell 命令"""
    try:
        result = subprocess.run(
            command,
            shell=True,
            capture_output=True,
            text=True,
            timeout=timeout,  # 超时限制
            encoding='utf-8',
            errors='ignore'
        )
        # ...
    except subprocess.TimeoutExpired:
        return f"⏱️ 命令执行超时（>{timeout}秒）"
```

**生活化类比**：就像给你的任务设置了一个"闹钟",如果闹钟响了任务还没完成,就强制停止,防止一直等待。

**安全机制总结表**：

| 安全机制 | 作用 | 示例 |
|---------|------|------|
| **命令黑名单** | 阻止危险的 Shell 命令 | 阻止 `rm -rf /` |
| **模块黑名单** | 阻止危险的 Python 模块 | 阻止 `os.system` |
| **文件操作检查** | 阻止危险的文件操作 | 阻止 `shutil.rmtree` |
| **超时限制** | 防止长时间运行 | 30秒超时 |

### 为什么需要安全机制？

**问题背景**：
- AI 模型可能会生成危险的命令或代码
- 用户可能无意中触发危险操作
- 工具执行可能破坏系统或数据

在工具执行前进行安全检查,拦截危险操作。

## 🔧 工具扩展机制

### MCP 工具支持

#### 专业定义
MCP (Model Context Protocol) 是一种协议,允许 Agent 连接外部工具服务器,扩展工具能力。通过 MCP,可以使用远程服务器提供的工具,而无需在本地实现。

#### 通俗解释与类比
想象你的工具箱不够用了,但是隔壁有个专门的工具店（MCP 服务器）。你可以:
1. 告诉店员你要用什么工具（指定 MCP 工具）
2. 店员把工具递给你（工具调用）
3. 你用完后把工具还回去（不需要本地存储）

#### 在项目中的具体体现

**MCP 中间件**：

```python
# 文件: ollamapilot/tools/mcp_tools.py

class MCPMiddleware(AgentMiddleware):
    """MCP 服务器中间件"""
    
    def __init__(
        self,
        server_url: str,
        server_name: str,
        allowed_tools: Optional[List[str]] = None,
        headers: Optional[Dict[str, str]] = None
    ):
        """
        初始化 MCP 中间件
        
        Args:
            server_url: MCP 服务器 URL
            server_name: 服务器名称标识
            allowed_tools: 允许使用的工具列表,None 表示全部
            headers: 请求头（如认证信息）
        """
        self.server_url = server_url
        self.server_name = server_name
        self.allowed_tools = allowed_tools
        self.headers = headers or {}
    
    def before_model(self, state: Any, runtime: Any) -> Optional[Dict[str, Any]]:
        """在模型调用前注入 MCP 配置"""
        mcp_config = {
            "type": "mcp",
            "server_url": self.server_url,
            "server_name": self.server_name,
            "allowed_tools": self.allowed_tools,
        }
        
        return {
            "metadata": {
                "mcp_servers": [mcp_config]
            }
        }
```

**使用示例**：

```python
from ollamapilot import create_agent, create_mcp_middleware

# 创建 MCP 中间件
mcp_mw = create_mcp_middleware(
    server_url="https://mcp.example.com/mcp",
    server_name="my-server",
    allowed_tools=["search", "query"]
)

# 创建 Agent 并添加 MCP 中间件
agent = create_agent(model, middleware=[mcp_mw])

# 现在 Agent 可以使用 MCP 服务器提供的工具
response = agent.invoke("搜索关于 AI 的最新新闻")
```

```python
# 文件: ollamapilot/tools/custom/__init__.py

def load_custom_tool(tool_path: str) -> Optional[BaseTool]:
    """
    加载自定义工具
    
    Args:
        tool_path: 工具路径,格式为 "filename.py:function_name"
        
    Returns:
        工具实例或 None
        
    Example:
        >>> tool = load_custom_tool("my_tool.py:my_function")
    """
    if ":" not in tool_path:
        return None
    
    file_name, func_name = tool_path.split(":", 1)
    
    # 构建完整路径
    custom_dir = Path(__file__).parent
    file_path = custom_dir / file_name
    
    if not file_path.exists():
        return None
    
    # 动态导入模块
    import importlib.util
    spec = importlib.util.spec_from_file_location(
        f"custom.{file_name[:-3]}", 
        file_path
    )
    module = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(module)
    
    # 获取工具函数
    tool_func = getattr(module, func_name, None)
    if tool_func is None:
        return None
    
    # 如果是 BaseTool 实例,直接返回
    if isinstance(tool_func, BaseTool):
        return tool_func
    
    # 如果是函数,转换为工具
    from langchain_core.tools import tool
    if callable(tool_func):
        return tool(tool_func)
    
    return None
```

## 🎯 总结：工具系统的核心优势

### 1. 模块化设计
- **工具函数独立**：每个工具都是独立函数,易于开发和测试
- **装饰器自动转换**：`@tool` 装饰器自动生成工具描述,无需手动编写 Schema
- **统一注册机制**：所有工具通过统一的注册机制管理,便于扩展

### 2. 安全可靠
- **多层安全检查**：命令黑名单、代码检查、超时限制,三重保护
- **自动重试机制**：工具调用失败时自动重试,提高成功率
- **调用次数限制**：防止无限循环,保证系统稳定

### 3. 高度可扩展
