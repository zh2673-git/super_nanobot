## 🎯 第一步：源码结构概览

OllamaPilot 的源码就像一个精密的机械钟表，每个模块都是独立的齿轮，协同工作：

- **ollamapilot/agent.py**：总调度器，像钟表的主发条和齿轮系统（协调所有组件、管理对话流程）
- **ollamapilot/skill_middleware.py**：中间件系统，像钟表的擒纵机构（控制执行节奏、插入处理逻辑）
- **ollamapilot/models.py**：模型管理，像钟表的动力源（连接 Ollama、初始化模型）
- **ollamapilot/skills/**：Skill 系统，像钟表的功能模块（独立车间、即插即用）
- **ollamapilot/tools/builtin.py**：内置工具，像钟表的指针和刻度（具体功能实现）

**为什么这样组织？**

传统项目的问题：
- 所有代码堆在一起，难以维护
- 修改一处可能影响其他部分
- 添加新功能需要改核心代码

OllamaPilot 的模块化设计：
- **职责分离**：每个文件负责一个明确的功能
- **低耦合**：模块之间通过接口交互，互不影响
- **易扩展**：添加新功能只需增加新模块

### 一、agent.py - Agent 核心

**专业定义**：Agent 是 OllamaPilot 的核心调度器，基于 LangChain 的 create_agent 工厂函数创建，负责初始化所有组件、管理对话流程、协调整个系统。

**通俗解释与类比**：
这就像公司的总经理办公室：
- 接待来访的客户（接收用户查询）
- 调度合适的部门处理（选择 Skill）
- 协调各个部门工作（调用工具）
- 记录重要会议内容（保存对话记忆）
- 整理结果回复客户（返回最终结果）

```python
class OllamaPilotAgent:
    """Agent 核心类"""
    
    def __init__(self, model, skills_dir, enable_memory, ...):
        """
        初始化 Agent
        
        就像公司开业前的准备工作：
        1. 注册各个部门（加载 Skill）
        2. 准备工具库（加载内置工具）
        3. 设置会议记录系统（配置记忆）
        4. 安排工作流程（配置中间件）
        """
        # 初始化 Skill 注册中心（注册各部门）
        self.skill_registry = SkillRegistry()
        if skills_dir:
            count = self.skill_registry.discover_skills(skills_dir)
            if self.verbose:
                print(f"📦 已加载 {count} 个 Skill")
        
        # 收集所有内置工具（准备工具库）
        self.builtin_tools = self._get_builtin_tools()
        
        # 配置记忆系统（设置会议记录）
        if enable_memory:
            self.checkpointer = MemorySaver()
        else:
            self.checkpointer = None
        
        # 构建中间件流水线（安排工作流程）
        middleware = self._build_middleware(max_tool_calls)
        
        # 创建 LangChain Agent（正式开业）
        self.agent = lc_create_agent(
            model=model.bind_tools(self.builtin_tools),
            tools=self.builtin_tools,
            middleware=middleware,
            checkpointer=self.checkpointer,
        )
```

```python
    limit_mw = ToolCallLimitMiddleware(run_limit=max_tool_calls)
    middleware.append(limit_mw)
```

```python
# agent.py：156-194 行
def invoke(self, query: str, thread_id: Optional[str] = None) -> str:
    """
    执行用户查询
    
    流程详解：
    1. 显示用户输入（如果启用 verbose）
    2. 手动选择 Skill 并显示激活日志
    3. 配置对话 ID
    4. 调用 Agent 执行
    5. 提取最终回复
    """
    # 步骤1：显示用户输入
    if self.verbose:
        print(f"🤖 用户: {query}")
    
    # 步骤2：选择 Skill（在 invoke 前显示）
    skill = self._select_skill_for_query(query)
    if skill and self.verbose:
        print(f"🎯 激活 Skill: {skill.name}")
    
    # 步骤3：配置对话 ID
    config = {"configurable": {"thread_id": thread_id or "default"}}
    
    # 步骤4：调用 Agent
    result = self.agent.invoke(
        {"messages": [HumanMessage(content=query)]},
        config
    )
    
    # 步骤5：提取回复
    messages = result.get("messages", [])
    if messages:
        last_message = messages[-1]
        if hasattr(last_message, "content"):
            response = last_message.content
            if self.verbose:
                print(f"🤖 AI: {response[:200]}{'...' if len(response) > 200 else ''}")
            return response
    
    return ""
```

```python
# agent.py：196-217 行
def _select_skill_for_query(self, query: str) -> Optional[Any]:
    """
    根据查询选择合适的 Skill
    
    选择流程：
    1. 查找匹配的特定 Skill
    2. 如果没有匹配，返回默认 Skill
    
    生活类比：
    就像公司前台接到客户电话：
    - 先查询是否有专门部门处理（查找匹配）
    - 如果没有专门的，转接综合部门（返回默认）
    """
    # 1. 查找匹配的特定 Skill
    matches = self.skill_registry.find_skill_by_trigger(query)
    
    if matches:
        return self.skill_registry.get_skill(matches[0])
    
    # 2. 返回默认 Skill
    default_skill = self.skill_registry.get_default_skill()
    if default_skill:
        return default_skill
    
    return None
```

```python
# registry.py：222-250 行
def find_skill_by_trigger(self, query: str) -> List[str]:
    """
    根据触发词查找 Skill
    
    算法：
    1. 将查询转为小写（不区分大小写）
    2. 遍历所有 Skill 的触发词
    3. 检查触发词是否在查询中
    4. 返回所有匹配的 Skill 名称
    
    示例：
    query = "明天苏州天气怎么样"
    
    遍历 Skill：
    - Weather Skill: 触发词 ["天气", "温度", "下雨"]
      - "天气" in "明天苏州天气怎么样" → ✓ 找到
      - 返回 "weather"
      
    - Python Skill: 触发词 ["python", "代码", "执行"]
      - 都不在查询中
      - 继续
      
    - File Skill: 触发词 ["文件", "读取", "写入"]
      - 都不在查询中
      - 继续
      
    最终返回：["weather"]
    """
    matches = []
    query_lower = query.lower()
    
    # 检查 Python Skill
    for skill_class in self._skill_classes.values():
        temp = skill_class()
        for trigger in temp.triggers:
            if trigger.lower() in query_lower:
                matches.append(temp.name)
                break  # 找到一个就足够
    
    # 检查 Markdown Skill
    for skill in self._markdown_skills.values():
        for trigger in skill.triggers:
            if trigger.lower() in query_lower:
                matches.append(skill.name)
                break
    
    return matches
```

### 二、skill_middleware.py - 中间件系统

**专业定义**：中间件系统实现了 LangChain 的 AgentMiddleware 接口，在 Agent 执行的不同阶段插入自定义逻辑，实现 Skill 选择、工具调用日志、自动重试等功能。

```python
class SkillSelectorMiddleware(AgentMiddleware):
    """
    Skill 选择器中间件
    
    职责：
    1. 根据用户查询选择合适的 Skill
    2. 注入 Skill 的系统提示词
    
    生活类比：
    就像公司的调度员：
    - 接到客户请求（before_model）
    - 根据请求内容判断应该由哪个部门处理（选择 Skill）
    - 给出工作指导（注入系统提示词）
    """
    
    def __init__(
        self,
        skill_registry: Any,
        default_skill_name: Optional[str] = "default",
        verbose: bool = False
    ):
        self.registry = skill_registry
        self.default_skill_name = default_skill_name
        self.verbose = verbose
        self._active_skill: Optional[str] = None
    
    def before_model(self, state: Any, runtime: Any) -> Optional[Dict[str, Any]]:
        """
        在模型调用前注入 Skill 的系统提示词
        
        执行时机：
        - Agent.invoke() 调用时，在模型推理之前
        
        流程：
        1. 获取最后一条用户消息
        2. 根据消息内容选择 Skill
        3. 获取 Skill 的系统提示词
        4. 检查是否已存在系统消息
        5. 如果没有，添加系统消息到消息列表开头
        
        返回状态更新：
        - None：不更新状态（已有系统消息或没有匹配的 Skill）
        - {"messages": [SystemMessage, ...messages]}：更新消息列表
        """
        # 步骤1：获取最后一条用户消息
        messages = state.get("messages", [])
        if not messages:
            return None
        
        last_message = messages[-1]
        query = ""
        if hasattr(last_message, "content"):
            query = str(last_message.content)
        
        if not query:
            return None
        
        # 步骤2：查找匹配的 Skill
        skill = self._select_skill(query)
        
        if skill:
            # 步骤3：获取系统提示词
            system_prompt = skill.get_system_prompt()
            if system_prompt:
                # 步骤4：检查是否已存在系统消息
                has_system = False
                for msg in messages:
                    if isinstance(msg, SystemMessage):
                        has_system = True
                        break
                
                if not has_system:
                    # 步骤5：添加系统消息到开头
                    return {
                        "messages": [SystemMessage(content=system_prompt)] + messages
                    }
        
        return None
    
    def _select_skill(self, query: str) -> Optional[Skill]:
        """
        根据查询选择合适的 Skill
        
        与 agent.py 中的 _select_skill_for_query 类似
        但这里是在中间件层面执行
        """
        matches = self.registry.find_skill_by_trigger(query)
        
        if matches:
            return self.registry.get_skill(matches[0])
        
        if self.default_skill_name:
            return self.registry.get_skill(self.default_skill_name)
        
        return None
```

```python
class ToolLoggingMiddleware(AgentMiddleware):
    """
    工具调用日志中间件

    记录工具调用的详细信息，便于调试

    - 记录工序使用的参数
    - 记录工序的执行结果
    - 记录工序的结束时间
    """

        self.verbose = verbose
    
    def wrap_tool_call(self, request: Any, handler: Any) -> Any:
        """
        包装工具调用，添加日志
        
        执行时机：
        - 每次工具调用时，在调用前和调用后
        
        流程：
        1. 提取工具名称和参数
        2. 记录调用前的日志
        3. 执行工具
        4. 记录调用后的日志
        5. 返回结果
        """
        if self.verbose:
            # 步骤1：提取工具调用信息
            tool_call = getattr(request, "tool_call", {})
            tool_name = tool_call.get("name", "unknown") if isinstance(tool_call, dict) else getattr(tool_call, "name", "unknown")
            tool_args = tool_call.get("args", {}) if isinstance(tool_call, dict) else getattr(tool_call, "args", {})
            
            # 步骤2：记录调用前
            print(f"🔧 执行工具: {tool_name}({tool_args})")
        
        # 步骤3：执行工具
        result = handler(request)
        
        if self.verbose:
            # 步骤4：记录调用后
            result_preview = str(result)[:200]
            if len(str(result)) > 200:
                result_preview += "..."
            print(f"   ✅ 结果: {result_preview}")
        
        # 步骤5：返回结果
```

```python
def discover_skills(self, skills_dir: str) -> int:
    """
    从目录自动发现 Skill
    
    流程：
    1. 检查目录是否存在
    2. 遍历所有子目录
    3. 对每个子目录：
       - 查找 skill.py（Python Skill）
       - 查找 SKILL.md（Markdown Skill）
       - 加载并注册
    
    生活类比：
    就像公司的部门招聘流程：
    1. 确认招聘地点（检查目录）
    2. 遍历所有应聘者（遍历子目录）
    3. 审查应聘材料（查找文件）
    4. 验证资格（加载并注册）
    
    返回：发现的 Skill 数量
    """
    skills_path = Path(skills_dir)
    if not skills_path.exists():
        print(f"⚠️ Skill 目录不存在: {skills_dir}")
        return 0
    
    count = 0
    
    # 遍历所有子目录
    for item in skills_path.iterdir():
        if item.is_dir() and not item.name.startswith("__"):
            skill_file = item / "skill.py"
            skill_md = item / "SKILL.md"
            
            if skill_file.exists():
                # 加载 Python Skill
                try:
                    self._load_skill_module(skill_file, item.name)
                    count += 1
                except Exception as e:
                    print(f"❌ 加载 Skill '{item.name}' 失败: {e}")
            
            elif skill_md.exists():
                # 加载 Markdown Skill
                try:
                    skill = load_markdown_skill(item)
                    if skill:
                        self.register_markdown_skill(skill)
                        count += 1
                except Exception as e:
                    print(f"❌ 加载 Markdown Skill '{item.name}' 失败: {e}")
    
    return count
```

```python
# registry.py：184-220 行
def _load_skill_module(self, file_path: Path, module_name: str) -> None:
    """
    动态加载 Skill 模块
    
    流程：
    1. 添加 Skill 目录到 Python 路径
    2. 动态加载模块
    3. 查找 Skill 类
    4. 注册到 _skill_classes
    
    生活类比：
    就像招聘新员工并分配工位：
    1. 分配工位（添加到 Python 路径）
    2. 办理入职手续（动态加载）
    3. 领取工作证（查找 Skill 类）
    4. 记录到员工表（注册）
    """
    # 步骤1：添加 Skill 目录到 Python 路径
    skill_dir = file_path.parent
    if str(skill_dir) not in sys.path:
        sys.path.insert(0, str(skill_dir))
    
    # 步骤2：动态加载模块
    spec = importlib.util.spec_from_file_location(
        f"skill_{module_name}",
        file_path
    )
    if spec is None or spec.loader is None:
        raise ImportError(f"无法加载模块: {file_path}")
    
    module = importlib.util.module_from_spec(spec)
    sys.modules[f"skill_{module_name}"] = module
    spec.loader.exec_module(module)
    
    # 步骤3：查找 Skill 类
    found = False
    for attr_name in dir(module):
        attr = getattr(module, attr_name)
        if (isinstance(attr, type) and
            issubclass(attr, Skill) and
            attr is not Skill):
            # 步骤4：注册
            self.register(attr)
            found = True
    
    if not found:
        raise ValueError(f"在 {file_path} 中未找到 Skill 类")
```
