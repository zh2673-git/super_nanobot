# OllamaPilot 三层搜索架构详解

> v0.2.5 核心特性：从基础搜索到深度研究的完整搜索体系

---

## 🎯 第一步：认识三层搜索架构的核心概念

OllamaPilot 的三层搜索架构就像一个"智能化搜索金字塔"，由三层能力递进的搜索服务组成：

- **第一层：基础搜索（web_search）**：像搜索引擎的"首页搜索框"，简单快速，适合日常查询（如查询天气、新闻、简单信息）
- **第二层：增强搜索（enhanced_search）**：像"专业资料库搜索"，支持多种专业引擎（如学术论文、代码仓库、百科知识）
- **第三层：深度研究（deep_research）**：像"专业研究助手"，多轮迭代、信息整合、生成报告（如复杂主题研究）

**为什么需要三层架构？**

想象你要查询信息：
- **简单的问题**："明天苏州天气" → 用第一层基础搜索，快速得到答案
- **专业的问题**："量子计算最新论文" → 用第二层增强搜索，从学术数据库搜索
- **深度的问题**："人工智能在医疗领域的应用现状和发展趋势" → 用第三层深度研究，生成完整研究报告

每层解决不同层次的需求，避免"用大炮打蚊子"的资源浪费。

## 📖 第三步：深入理解每一层

### 第一层：基础搜索（web_search）

**专业定义**：基于 SearXNG 的聚合搜索引擎，通过 Docker 自动部署，实现零配置的网络搜索能力。

**通俗解释与类比**：
这就像搜索引擎的首页搜索框：
- 输入任何关键词都能得到结果
- 不需要注册、不需要 API Key
- 自动聚合百度、Google、必应等多个引擎的结果
- 就像你有一个"搜索引擎超市"，一次搜索问遍多个引擎

**核心组成**：

1. **SearXNG（搜索引擎）**：
   - 开源元搜索引擎
   - 聚合 70+ 搜索源
   - 保护隐私（不追踪用户）

2. **Docker 容器**：
   - 自动拉取镜像
   - 一键启动服务
   - 端口映射：8080

3. **自动部署机制**：
   ```
   [检查服务] → [未运行] → [检查Docker] → [拉取镜像] → [启动容器]
                                        ↓
                                    [已运行] → [直接使用]
   ```

**在项目中的具体体现**：

Skill 配置（`skills/web_search/SKILL.md`）：

```bash
# 4. 等待服务就绪

```

### 第二层：增强搜索（enhanced_search）

**专业定义**：多引擎专业搜索系统，支持学术文献、代码仓库、百科知识等多种专业领域的搜索，满足不同场景的专业信息获取需求。

**通俗解释与类比**：
这就像一个"专业资料库搜索系统"：
- 学术论文 → PubMed（医学文献数据库）
- 开源代码 → GitHub（代码仓库）
- 百科知识 → 百度百科（中文百科）
- 综合信息 → 多引擎聚合

每种搜索都有专业的引擎支持，就像有不同的"专家"帮你找不同类型的信息。

**核心组成**：

1. **搜索引擎工厂（Engine Factory）**：
   ```python
   # engines/base.py
   class SearchEngine:
       def search(self, query: str, num_results: int) -> List[SearchResult]:
           """搜索引擎基类"""
           pass
   ```

2. **已实现引擎**：
   | 引擎 | 类别 | 文件 | 状态 |
   |------|------|------|------|
   | SearXNG | 通用 | `engines/searxng.py` | ✅ 可用 |
   | PubMed | 学术 | `engines/pubmed.py` | ✅ 可用 |
   | 百度百科 | 百科 | `engines/baidu_baike.py` | ✅ 可用 |
   | GitHub | 代码 | `engines/github.py` | ✅ 可用 |
   | arXiv | 学术 | `engines/arxiv.py` | ⚠️ 受限 |
   | Wikipedia | 百科 | `engines/wikipedia.py` | ⚠️ 受限 |

3. **聚合器（Aggregator）**：
   ```python
   # aggregator.py
   class SearchAggregator:
       def aggregate(self, results: List[SearchResult]) -> List[SearchResult]:
           """聚合多引擎结果，去重排序"""
           pass
   ```

### 第三层：深度研究（deep_research）

**专业定义**：基于多轮迭代的智能研究系统，自动搜索、整合、分析信息，生成结构化的 Markdown 研究报告。

**通俗解释与类比**：
这就像一个"专业研究助手"：
- 你告诉它一个研究主题
- 它会自动规划研究步骤
- 多次搜索、阅读、整合信息
- 最终生成一份完整的研究报告

就像你请了一个研究生帮你查资料写报告，但这个助手：
- 不知疲倦（可以反复搜索）
- 信息全面（使用多个搜索引擎）
- 结构清晰（生成专业报告）
- 永久保存（自动保存到文件）

**核心组成**：

1. **研究流程（Research Workflow）**：
   ```
   [需求澄清] → [研究简报] → [多轮搜索] → [结果整合] → [报告生成]
   ```

2. **状态管理（State Management）**：
   ```python
   # state.py
   class ResearchState:
       topic: str              # 研究主题
       findings: List[str]    # 已发现的信息
       sources: List[str]     # 信息来源
       iterations: int        # 当前迭代次数
       max_iterations: int    # 最大迭代次数
   ```

3. **报告生成（Report Generation）**：
   ```python
   def generate_report(state: ResearchState) -> str:
       """生成 Markdown 格式的研究报告"""
       report = f"""# {state.topic} 研究报告
       
       ## 研究概述
       {state.summary}
       
       ## 关键发现
       {state.findings}
       
       ## 参考来源
       {state.sources}
       """
       return report
   ```

## 🔧 第四步：使用指南

### 基础搜索使用

```python
from ollamapilot import init_ollama_model, create_agent

model = init_ollama_model("qwen3.5:4b")
agent = create_agent(model, skills_dir="skills")

# 基础搜索
agent.invoke("搜索 Python 教程")
agent.invoke("查一下北京天气")
agent.invoke("找最新的 AI 新闻")
```

### 增强搜索使用

```python
# 学术搜索 - PubMed 医学文献
agent.invoke("增强搜索 量子计算论文")
agent.invoke("增强搜索 肿瘤治疗最新研究")

# 代码搜索 - GitHub 开源项目
agent.invoke("增强搜索 Python 爬虫框架")
agent.invoke("增强搜索 machine learning 项目")

# 百科搜索 - 百度百科
agent.invoke("增强搜索 什么是区块链")
agent.invoke("增强搜索 量子力学定义")

# 综合搜索 - 多引擎聚合
agent.invoke("增强搜索 深度学习应用")
agent.invoke("增强 search 自然语言处理")
```

### 深度研究使用

```python
# 深度研究自动生成报告
agent.invoke("研究 人工智能在医疗诊断中的应用")
```

## 💡 第五步：最佳实践

### 选择合适的搜索层级

| 需求类型 | 推荐层级 | 示例 |
|---------|---------|------|
| 快速查询 | 第一层 | "明天天气"、"Python 教程" |
| 专业资料 | 第二层 | "医学论文"、"开源项目" |
| 深度研究 | 第三层 | "AI发展趋势"、"技术报告" |

### 网络限制说明

| 引擎 | 状态 | 说明 |
|------|------|------|
| SearXNG | ✅ 可用 | 本地部署，优先使用 |
| PubMed | ✅ 可用 | 医学文献，国内可访问 |
| 百度百科 | ✅ 可用 | 中文百科，国内可访问 |
| GitHub | ✅ 可用 | 代码仓库，国内可访问 |
| arXiv | ⚠️ 受限 | 国际论文，可能需代理 |
| Wikipedia | ⚠️ 受限 | 国际百科，可能需代理 |

### 性能优化建议

1. **基础搜索**：适合简单查询，响应快
2. **增强搜索**：根据需求选择引擎，避免使用不可用引擎
3. **深度研究**：设置合理的迭代次数（默认6次），避免过度搜索
