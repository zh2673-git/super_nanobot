# OllamaPilot Web 搜索部署详解

> **创建时间**: 2026-03-09  
> **项目**: OllamaPilot - 基于 LangChain 1.0+ 的本地智能助手框架  
> **核心特性**: 完全本地运行的网络搜索能力，基于 SearXNG + Docker

---

## 🧩 第一步：点出 Web 搜索部署的组件概念

OllamaPilot 的 Web 搜索功能由以下几个核心组件构成，每个组件都有明确的职责：

- **SearXNG（搜索引擎）**：一个开源的元搜索引擎，聚合多个搜索引擎的结果（就像一个"搜索引擎超市"，一次搜索能同时问百度、Google、必应等多个引擎）
- **Docker（容器平台）**：用于运行 SearXNG 的容器化平台（就像一个"便携式虚拟机"，把软件和它需要的所有东西打包在一起，随处运行）
- **Docker 镜像（Image）**：SearXNG 的打包文件（就像软件的"安装盘"，包含了运行 SearXNG 所需的一切）
- **Docker 容器（Container）**：运行中的 SearXNG 实例（就像从"安装盘"安装后正在运行的软件）
- **自动部署脚本（Auto Deployment）**：OllamaPilot 内置的智能部署逻辑（就像一个"一键安装助手"，自动检测环境、拉取镜像、启动服务）

## 📖 第三步：详细解释每个组件

### 一、SearXNG（开源元搜索引擎）

#### 专业定义

SearXNG 是一个基于 Python 的开源元搜索引擎（Metasearch Engine），它本身不维护索引，而是将用户的搜索请求转发给多个搜索引擎，然后将结果聚合、去重、排序后返回给用户。

#### 通俗解释与类比

想象你要买东西，想比较多家店的价格：
- **传统方式**：挨个去京东、淘宝、拼多多搜索，手动比较
- **SearXNG 方式**：告诉 SearXNG 你要买什么，它帮你同时去多家店问价，然后把所有结果整理好给你

**SearXNG 就像一个"购物比价助手"**：
- 它不自己卖东西（不维护索引）
- 它帮你去各大平台问价（转发搜索请求）
- 它把结果整理好给你（聚合、排序）

#### 为什么选择 SearXNG？

| 特性 | SearXNG | Google API | Bing API |
|------|---------|------------|----------|
| **成本** | 完全免费 | 需要付费 | 需要付费 |
| **隐私** | 不追踪用户 | 追踪用户数据 | 追踪用户数据 |
| **限制** | 无请求限制 | 有请求配额 | 有请求配额 |
| **数据来源** | 聚合多个引擎 | 只有 Google | 只有 Bing |
| **部署方式** | 本地部署 | 云端调用 | 云端调用 |

#### 在项目中的具体体现

**SearXNG 配置**：
```python
# 文件: ollamapilot/tools/builtin.py

# SearXNG 默认配置
SEARXNG_URL = os.environ.get("SEARXNG_URL", "http://localhost:8080")

# 搜索请求参数
params = {
    "q": query,           # 搜索关键词
    "format": "json",     # 返回 JSON 格式
    "language": "zh-CN",  # 中文语言
    "safesearch": "0",    # 关闭安全搜索
}
```

**SearXNG 工作原理**：
```
[用户搜索 "Python 教程"]
    |
    v
[SearXNG 解析请求]
    |
    +---> 转发给 Google   ---> 获取结果
    +---> 转发给 Bing     ---> 获取结果
    +---> 转发给 DuckDuckGo ---> 获取结果
    +---> 转发给 Wikipedia ---> 获取结果
    |
    v
[聚合结果]
    |
    +---> 去重（移除重复链接）
    +---> 排序（按相关性）
    +---> 格式化（JSON 输出）
    |
    v
[返回给 OllamaPilot]
```

### 二、Docker（容器化平台）

#### 专业定义

Docker 是一个开源的容器化平台，它使用 Linux 内核的特性（如 cgroups 和 namespaces）来创建独立、隔离的运行环境，称为"容器"。容器共享宿主机的操作系统内核，但拥有独立的文件系统、网络、进程空间。

#### 通俗解释与类比

**传统软件安装 vs Docker 容器**：

想象你要在电脑上安装一个复杂的软件：

**传统方式（手动安装）**：
```
1. 下载软件安装包
2. 检查系统是否安装了依赖（Python、Node.js 等）
3. 如果没装，先安装依赖
4. 如果依赖版本不对，可能会冲突
5. 安装软件
6. 配置环境变量
7. 如果出问题，很难排查是哪里错了
```

**Docker 方式（容器化）**：
```
1. 运行一条命令：docker run xxx
2. 完成！（Docker 自动处理所有依赖、配置）
```

**生活类比**：

- **传统安装** = 你自己盖房子（买砖、调水泥、安装水电...）
- **Docker 容器** = 你直接搬进一个装修好的移动房屋（所有设施都准备好了）

**Docker 核心概念对比**：

| 概念 | 生活类比 | 技术含义 |
|------|---------|----------|
| **镜像 (Image)** | 安装光盘 / 预制房屋 | 只读的文件系统和配置，不可变 |
| **容器 (Container)** | 运行中的软件 / 入住的房屋 | 镜像的运行实例，可读写 |
| **仓库 (Registry)** | 软件商店 / 房屋展示中心 | 存储和分发镜像的地方 |
| **Dockerfile** | 房屋设计图纸 | 定义如何构建镜像的脚本 |
| **Docker Engine** | 房屋建造工具 / 施工队 | 运行容器的核心引擎 |

#### Docker 的核心优势

1. **一致性**：开发、测试、生产环境完全一致
   - 就像你搬家的移动房屋，在哪都是一样的

2. **隔离性**：容器之间互不影响
   - 就像每栋移动房屋都有独立的墙壁、水电

3. **轻量级**：比虚拟机更轻量
   - 虚拟机 = 整栋大楼（包含独立的电梯、水电系统）
   - 容器 = 一间公寓（共享大楼的基础设施）

4. **快速启动**：毫秒级启动
   - 虚拟机启动需要几分钟
   - 容器启动只需要几秒

### 三、Docker 部署流程详解

#### 专业定义

Docker 部署流程包括：安装 Docker Engine、拉取镜像、创建容器、配置网络和存储、启动服务、健康检查。

#### 通俗解释与类比

**Docker 部署就像买房入住**：

1. **安装 Docker** = 盖好小区（搭建基础设施）
2. **拉取镜像** = 从房屋展示中心选择房型（下载预制房屋）
3. **创建容器** = 把房屋运到指定位置（初始化容器）
4. **配置端口映射** = 安装门牌号（设置网络访问）
5. **启动服务** = 接通水电，入住（启动应用）
6. **健康检查** = 定期检查房屋是否正常运转（监控服务状态）

#### 在项目中的具体体现

**完整部署流程代码分析**：

```python
# 文件: ollamapilot/tools/builtin.py

def _start_searxng_docker() -> tuple[bool, str]:
    """
    自动启动 SearXNG Docker 容器
    
    完整流程：
    1. 检查 Docker 是否安装
    2. 检查是否已有容器在运行
    3. 检查是否有停止的容器
    4. 检查镜像是否存在（不存在则自动拉取）
    5. 创建并启动容器
    6. 等待服务就绪
    """
    try:
        # ===== 步骤 1: 检查 Docker 是否安装 =====
        result = subprocess.run(
            ["docker", "--version"],
            capture_output=True,
            text=True,
            timeout=10
        )
        if result.returncode != 0:
            return False, "Docker 未安装或未启动"
        
        # ===== 步骤 2: 检查是否已有容器在运行 =====
        result = subprocess.run(
            ["docker", "ps", "-q", "-f", "name=searxng"],
            capture_output=True,
            text=True,
            timeout=10
        )
        
        if result.stdout.strip():
            # 容器已在运行，检查健康状态
            if _check_searxng_running():
                return True, "SearXNG 容器已在运行"
            else:
                return False, "SearXNG 容器存在但无法访问"
        
        # ===== 步骤 3: 检查是否有停止的容器 =====
        result = subprocess.run(
            ["docker", "ps", "-aq", "-f", "name=searxng"],
            capture_output=True,
            text=True,
            timeout=10
        )
        
        if result.stdout.strip():
            # 启动已存在的容器
            subprocess.run(
                ["docker", "start", "searxng"],
                capture_output=True,
                timeout=30
            )
        else:
            # ===== 步骤 4: 检查镜像是否存在 =====
            if not _check_docker_image_exists("searxng/searxng"):
                success, message = _pull_docker_image("searxng/searxng")
                if not success:
                    return False, f"镜像拉取失败: {message}"
            
            # ===== 步骤 5: 创建新容器 =====
            subprocess.run(
                [
                    "docker", "run", "-d",
                    "--name", "searxng",
                    "-p", "8080:8080",
                    "-e", "BASE_URL=http://localhost:8080/",
                    "-e", "INSTANCE_NAME=OllamaPilot-SearXNG",
                    "searxng/searxng"
                ],
                capture_output=True,
                timeout=60
            )
        
        # ===== 步骤 6: 等待服务启动 =====
        for i in range(30):  # 最多等待30秒
            time.sleep(1)
            if _check_searxng_running():
                return True, "SearXNG 启动成功"
        
        return False, "SearXNG 启动超时"
        
    except subprocess.TimeoutExpired:
        return False, "Docker 命令执行超时"
    except FileNotFoundError:
        return False, "Docker 命令未找到"
    except Exception as e:
        return False, f"启动失败: {str(e)}"
```

**流程图详解**：

```
[开始部署]
    |
    v
[步骤1: 检查 Docker 安装]
    |-> docker --version
    |-> 成功 -> 继续
    |-> 失败 -> 返回 "Docker 未安装"
    |
    v
[步骤2: 检查运行中的容器]
    |-> docker ps -q -f name=searxng
    |-> 有容器运行 -> 检查健康状态
    |   |-> 健康 -> 返回 "已在运行"
    |   |-> 不健康 -> 返回错误
    |-> 无容器运行 -> 继续
    |
    v
[步骤3: 检查停止的容器]
    |-> docker ps -aq -f name=searxng
    |-> 有停止的容器 -> 启动它 (docker start searxng)
    |-> 无容器 -> 继续
    |
    v
[步骤4: 检查镜像是否存在]
    |-> docker images -q searxng/searxng
    |-> 镜像存在 -> 跳过拉取
    |-> 镜像不存在 -> 拉取镜像 (docker pull searxng/searxng)
    |   |-> 拉取成功 -> 继续
    |   |-> 拉取失败 -> 返回错误
    |
    v
[步骤5: 创建并启动容器]
    |-> docker run -d --name searxng -p 8080:8080 searxng/searxng
    |-> 创建成功 -> 继续
    |-> 创建失败 -> 返回错误
    |
    v
[步骤6: 等待服务就绪]
    |-> 循环检查 http://localhost:8080/healthz
    |-> 最多等待30秒
    |-> 成功 -> 返回 "启动成功"
    |-> 超时 -> 返回 "启动超时"
    |
    v
[结束]
```

### 四、Docker 核心原理详解

#### 专业定义

Docker 的核心原理基于 Linux 内核的三项技术：
1. **Namespaces（命名空间）**：实现资源隔离
2. **Cgroups（控制组）**：实现资源限制
3. **UnionFS（联合文件系统）**：实现分层存储

#### 通俗解释与类比

**1. Namespaces（命名空间）= 独立房间**

想象一栋大楼，每个房间（容器）都有：
- 自己的门牌号（PID namespace - 进程 ID）
- 自己的网络（Network namespace - 网络设备）
- 自己的文件系统（Mount namespace - 文件挂载）
- 自己的用户（User namespace - 用户权限）

每个房间的人以为整栋楼只有自己，看不到其他房间。

**生活类比**：
- Namespaces = 每个房间有独立的门、窗户、电话号码
- 容器内进程 = 房间里的人
- 宿主机进程 = 大楼里所有房间的人

**2. Cgroups（控制组）= 资源配额**

想象大楼管理员限制每个房间的资源：
- 房间 A 最多用电 100 度（CPU 限制）
- 房间 B 最多用水 50 吨（内存限制）
- 房间 C 最多用网速 10MB/s（网络限制）

**生活类比**：
- Cgroups = 每个房间有独立的电表、水表、网速限制
- CPU 限制 = 限制房间用电量
- 内存限制 = 限制房间用水量
- 磁盘限制 = 限制房间储物空间

**3. UnionFS（联合文件系统）= 分层存储**

想象一本画册，由多层透明纸组成：
- 底层是基础图画（基础镜像）
- 中间层是添加的颜色（软件包）
- 顶层是你的涂改（容器写入）

**生活类比**：
- UnionFS = 透明胶片叠加
- 只读层 = 印刷好的底图
- 可写层 = 你可以在上面涂改
- 多个容器共享只读层 = 多个人看同一本画册，各自涂改

#### 在项目中的具体体现

**Docker 分层存储示意**：

```
┌─────────────────────────────────────────────────────────┐
│                 SearXNG 容器文件系统                      │
├─────────────────────────────────────────────────────────┤
│  ┌────────────────────────────────────────────────────┐│
│  │  容器层（Container Layer - 可写）                   ││
│  │  - 运行时产生的日志文件                              ││
│  │  - 临时文件                                         ││
│  │  - 用户修改的配置                                   ││
│  └────────────────────────────────────────────────────┘│
│  ┌────────────────────────────────────────────────────┐│
│  │  应用层（Application Layer - 只读）                 ││
│  │  - SearXNG 应用代码                                 ││
│  │  - Python 依赖包                                    ││
│  │  - 配置文件模板                                     ││
│  └────────────────────────────────────────────────────┘│
│  ┌────────────────────────────────────────────────────┐│
│  │  运行时层（Runtime Layer - 只读）                   ││
│  │  - Python 解释器                                    ││
│  │  - 系统库                                          ││
│  │  - 网络工具                                         ││
│  └────────────────────────────────────────────────────┘│
│  ┌────────────────────────────────────────────────────┐│
│  │  基础层（Base Layer - 只读）                        ││
│  │  - Alpine/Ubuntu 基础系统                           ││
│  │  - 系统工具（sh, ls, cat 等）                       ││
│  │  - 基础库                                          ││
│  └────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘

优势：
1. 只读层可以被多个容器共享，节省空间
2. 修改只发生在最上层，不影响底层镜像
3. 构建镜像时可以复用已有层，加快构建速度
```

**Docker 网络模式示意**：

```
┌─────────────────────────────────────────────────────────────┐
│                    Docker 网络架构                           │

│  └───────────────────────────────────────────────────────┘ │

│         ┌───────────────┼───────────────┐                  │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 五、Web 搜索部署实践

#### 方式一：全自动部署（推荐）

OllamaPilot 内置了自动部署功能，首次使用时会自动完成所有步骤：

```python
from ollamapilot import init_ollama_model, create_agent

# 初始化模型
model = init_ollama_model("qwen3.5:4b")
agent = create_agent(model, skills_dir="skills")

# 首次搜索会自动部署 SearXNG
response = agent.invoke("搜索 Python 教程")
# 自动执行：
# 1. 检测 Docker 是否安装
# 2. 检测镜像是否存在（不存在则自动拉取）
# 3. 创建并启动容器
# 4. 等待服务就绪
# 5. 执行搜索
```

**自动部署的优势**：
- 零手动操作
- 自动检测环境
- 自动拉取镜像
- 自动启动服务
- 自动健康检查

#### 方式二：手动部署

如果需要手动控制部署过程：

```bash
# 步骤1: 拉取镜像
docker pull searxng/searxng

# 步骤2: 创建并启动容器
docker run -d \
  --name searxng \
  -p 8080:8080 \
  -e BASE_URL=http://localhost:8080/ \
  -e INSTANCE_NAME=OllamaPilot-SearXNG \
  searxng/searxng

# 步骤3: 检查容器状态
docker ps | grep searxng

# 步骤4: 测试服务
curl http://localhost:8080/healthz
```

**Docker 命令详解**：

| 参数 | 含义 | 说明 |
|------|------|------|
| `-d` | 后台运行 | 容器在后台运行，不会阻塞终端 |
| `--name searxng` | 容器名称 | 便于后续管理和引用 |
| `-p 8080:8080` | 端口映射 | 宿主机8080端口映射到容器8080端口 |
| `-e KEY=VALUE` | 环境变量 | 设置容器内的环境变量 |
| `searxng/searxng` | 镜像名称 | 使用官方 SearXNG 镜像 |

#### 方式三：使用远程 SearXNG

如果已有远程 SearXNG 服务，可以通过环境变量配置：

```python
import os

# 设置远程 SearXNG 地址
os.environ["SEARXNG_URL"] = "http://your-searxng-server:8080"

# 然后正常使用 web_search
from ollamapilot import init_ollama_model, create_agent

model = init_ollama_model("qwen3.5:4b")
agent = create_agent(model, skills_dir="skills")
response = agent.invoke("搜索 Python 教程")
```

### 七、故障排查指南

#### 常见问题及解决方案

**问题 1: Docker 未安装**

```
错误信息: Docker 未安装或未启动

解决方案:
1. 安装 Docker Desktop
   - Windows: https://www.docker.com/products/docker-desktop
   - Mac: https://www.docker.com/products/docker-desktop
   - Linux: sudo apt-get install docker-ce docker-ce-cli containerd.io

2. 启动 Docker 服务
   - Windows/Mac: 启动 Docker Desktop
   - Linux: sudo systemctl start docker

3. 验证安装
   docker --version
```

**问题 2: 端口被占用**

```
错误信息: Port 8080 is already in use

解决方案:
1. 查看端口占用
   - Windows: netstat -ano | findstr :8080
   - Mac/Linux: lsof -i :8080

2. 停止占用端口的进程

3. 或者使用其他端口
   docker run -d --name searxng -p 9090:8080 searxng/searxng
   然后设置环境变量: SEARXNG_URL=http://localhost:9090
```

**问题 3: 镜像拉取失败**

```
错误信息: 镜像拉取失败

解决方案:
1. 检查网络连接
   ping hub.docker.com

2. 配置 Docker 镜像加速器
   编辑 /etc/docker/daemon.json (Linux)
   或 Docker Desktop 设置 (Windows/Mac)

3. 手动拉取镜像
   docker pull searxng/searxng
```

**问题 4: 容器启动后无法访问**

```
错误信息: SearXNG 容器存在但无法访问

解决方案:
1. 检查容器状态
   docker ps -a | grep searxng

2. 查看容器日志
   docker logs searxng

3. 检查容器健康状态
   curl http://localhost:8080/healthz

4. 重启容器
   docker restart searxng
```

**问题 5: 搜索无结果**

```
错误信息: 未找到搜索结果

解决方案:
1. 检查 SearXNG 配置
   docker exec -it searxng cat /etc/searxng/settings.yml

2. 检查网络连接
   SearXNG 需要访问外部搜索引擎

3. 查看详细日志
   docker logs --tail 100 searxng
```

### 八、性能优化建议

#### 容器资源限制

```bash
# 限制 CPU 和内存使用
docker run -d \
  --name searxng \
  -p 8080:8080 \
  --cpus=1 \          # 限制使用 1 个 CPU 核心
  --memory=512m \     # 限制使用 512MB 内存
  searxng/searxng
```

#### 镜像优化

```bash
# 使用轻量级镜像
# SearXNG 官方镜像基于 Alpine，体积小，启动快

# 查看镜像大小
docker images searxng/searxng

# 清理未使用的镜像
docker image prune
```

#### 网络优化

```bash
# 使用 host 网络模式（性能更好，但安全性较低）
docker run -d \
  --name searxng \
  --network host \
  searxng/searxng

# 注意: host 模式下不需要 -p 端口映射
```
