# Local Deep Research 源码解析

> 本笔记深入分析项目的核心源码，帮助理解关键组件的实现细节。

---

## 第一步：点出核心代码组件概念

- **[研究服务（ResearchService）]：项目的"总调度员"，管理整个研究流程的启动、执行和结果保存，就像公司的"项目经理"**
- **[搜索系统（AdvancedSearchSystem）]：智能"搜索指挥官"，协调多种搜索引擎并优化搜索结果，就像"猎头公司"同时联系多个"候选人"**
- **[报告生成器（ReportGenerator）]：AI"作家"，把收集到的资料整理成可读的Markdown报告，就像"编辑"**
- **[数据库模型（Database Models）]：数据的"表格模板"，定义如何存储研究历史和用户信息，就像"excel表头"**
- **[LLM配置（LLM Config）]：AI模型的"遥控器"，管理不同AI模型的调用参数，就像"汽车钥匙"控制不同的"发动机"**

## 第三步：详细解释核心源码

### 1. 研究服务（ResearchService）

**专业定义**：研究服务的核心函数，负责管理整个研究流程的启动、执行和完成。

**通俗解释与类比**：就像公司的"项目经理"：
- 接收客户需求（研究请求）
- 制定工作计划（研究策略）
- 协调团队工作（搜索、分析）
- 交付最终成果（报告）
- 记录项目档案（保存到数据库）

**核心代码分析**：
```python
# 文件位置
src/local_deep_research/web/services/research_service.py

# 核心入口函数
@log_for_research
@thread_cleanup
def run_research_process(research_id, query, mode, **kwargs):
    """
    执行研究的主函数
    
    研究流程：
    1. 检查是否被终止
    2. 初始化研究引擎
    3. 执行搜索迭代
    4. 生成报告
    5. 保存结果到数据库
    """
    # 第一步：检查研究是否被终止
    if is_termination_requested(research_id):
        cleanup_research_resources(research_id, username)
        return
    
    # 第二步：提取关键参数
    model_provider = kwargs.get("model_provider")
    model = kwargs.get("model")
    search_engine = kwargs.get("search_engine")
    strategy = kwargs.get("strategy", "source-based")
    
    # 第三步：执行研究
    results = await search_system.research(query, strategy)
    
    # 第四步：生成报告
    report = await report_generator.generate(results)
    
    # 第五步：保存结果
    save_to_database(research_id, report)
```

**关键参数解析**：
| 参数 | 类型 | 说明 | 例子 |
|-----|------|------|------|
| research_id | str | 研究的唯一标识 | "abc123-def456" |
| query | str | 研究问题 | "量子计算最新进展" |
| mode | str | 研究模式 | "quick", "detailed" |
| model_provider | str | AI模型供应商 | "openai", "ollama" |
| model | str | 具体模型名 | "gpt-4", "llama3" |
| search_engine | str | 搜索引擎 | "searxng", "arxiv" |
| strategy | str | 研究策略 | "source-based" |

### 2. 搜索系统（AdvancedSearchSystem）

**专业定义**：高级搜索系统，负责协调多种搜索引擎，执行搜索查询，并对结果进行排序和过滤。

**通俗解释与类比**：就像"猎头公司"：
- 需要找人才时（搜索资料）
- 同时联系多个渠道（多个搜索引擎）
- 对候选人进行筛选和排序（结果排名）
- 推荐最合适的人选（最相关的结果）

**核心代码分析**：
```python
# 文件位置
src/local_deep_research/search_system/__init__.py

class AdvancedSearchSystem:
    """高级搜索系统"""
    
    def __init__(self, config):
        # 初始化搜索引擎工厂
        self.engine_factory = SearchEngineFactory()
        
        # 加载启用的搜索引擎
        self.engines = self._load_engines(config)
    
    async def research(self, query, strategy):
        """执行搜索研究"""
        results = []
        
        # 1. 为每个搜索引擎生成特定的查询
        queries = self._generate_queries(query, strategy)
        
        # 2. 并行执行所有搜索
        search_tasks = []
        for engine_name, search_query in queries.items():
            task = self._search_with_engine(engine_name, search_query)
            search_tasks.append(task)
        
        # 等待所有搜索完成
        all_results = await asyncio.gather(*search_tasks)
        
        # 3. 合并和排序结果
        merged = self._merge_results(all_results)
        ranked = self._rank_results(merged, query)
        
        return ranked
```

**搜索策略类型**：
| 策略名 | 说明 | 适用场景 |
|-------|------|---------|
| source-based | 基于来源的搜索 | 需要多来源验证 |
| focused_iteration | 聚焦迭代 | 深入分析特定主题 |
| broad_coverage | 广泛覆盖 | 需要全面了解 |
| academic_search | 学术搜索 | 学术研究 |

### 3. 报告生成器（ReportGenerator）

**专业定义**：将搜索结果和分析转化为结构化报告的模块，支持Markdown、PDF等多种格式。

**通俗解释与类比**：就像"出版社编辑"：
- 收集作者交来的稿子（LLM生成的内容）
- 检查引用是否准确（事实核查）
- 格式化排版（Markdown/PDF）
- 添加目录和章节（结构化）

**核心代码分析**：
```python
# 文件位置
src/local_deep_research/report_generator/__init__.py

class IntegratedReportGenerator:
    """综合报告生成器"""
    
    def __init__(self, llm, citation_formatter):
        self.llm = llm  # 大语言模型
        self.citation_formatter = citation_formatter  # 引用格式化器
    
    async def generate(self, research_results):
        """生成研究报告"""
        
        # 1. 准备上下文
        context = self._prepare_context(research_results)
        
        # 2. 构建prompt
        prompt = self._build_prompt(context)
        
        # 3. 调用LLM生成报告
        report_content = await self.llm.agenerate([prompt])
        
        # 4. 添加引用
        cited_report = self.citation_formatter.format(
            report_content, 
            research_results
        )
        
        # 5. 格式化输出
        formatted = self._format_markdown(cited_report)
        
        return formatted
    
    def _build_prompt(self, context):
        """构建生成报告的prompt"""
        return f"""
        你是一个专业的研究报告撰写助手。
        
        请根据以下研究结果撰写一份详细的研究报告：
        
        {context}
        
        要求：
        1. 结构清晰，包含摘要、主体、结论
        2. 每处引用来源都要标注
        3. 使用Markdown格式
        4. 语言专业但易懂
        """
```

### 4. 数据库模型（Database Models）

**专业定义**：使用SQLAlchemy定义的数据模型，包括用户、研究历史、研究策略等。

**通俗解释与类比**：就像"公司的人事表格"：
- 员工信息表（用户）
- 项目记录表（研究历史）
- 工作方法表（研究策略）

**核心模型定义**：
```python
# 文件位置

class ResearchHistory(db.Model):
    """研究历史模型"""
    __tablename__ = 'research_history'
    
    # 主键
    id = db.Column(db.Integer, primary_key=True)
    
    # 研究ID（唯一标识）
    research_id = db.Column(db.String(36), unique=True)
    
    # 用户
    username = db.Column(db.String(255))
    
    # 研究内容
    query = db.Column(db.Text)  # 研究问题
    result = db.Column(db.Text)  # 研究结果
    summary = db.Column(db.Text)  # 摘要

    status = db.Column(db.String(50))  # 状态

    completed_at = db.Column(db.DateTime)  # 完成时间
    
    # 关联
    sources = db.relationship('Source', backref='research')

    username = db.Column(db.String(255), unique=True)
```

### 5. LLM配置（LLM Config）

**专业定义**：管理不同大语言模型provider的配置和初始化的模块。

**通俗解释与类比**：就像"汽车钥匙"：
- 不同的钥匙（配置）对应不同的车（LLM Provider）
- 每把钥匙有自己的"密码"（API Key）
- 不同的车有不同的"操作方式"（接口）

**配置示例**：
```python
# 文件位置
src/local_deep_research/config/llm_config.py

def get_llm(provider: str, model: str, **kwargs):
    """
    获取LLM实例
    
    Args:
        provider: 模型供应商 ("openai", "anthropic", "ollama", etc.)
        model: 具体模型名
        **kwargs: 其他参数 (api_key, temperature, etc.)
    
    Returns:
        配置好的LLM实例
    """
    
    if provider == "openai":
        from langchain_openai import ChatOpenAI
        return ChatOpenAI(
            model=model,
            api_key=kwargs.get("api_key"),
            temperature=kwargs.get("temperature", 0.7),
            max_tokens=kwargs.get("max_tokens", 2000),
        )
    
    elif provider == "anthropic":
        from langchain_anthropic import ChatAnthropic
        return ChatAnthropic(
            model=model,
            anthropic_api_key=kwargs.get("api_key"),
            temperature=kwargs.get("temperature", 0.7),
        )
    
    elif provider == "ollama":
        from langchain_ollama import ChatOllama
        return ChatOllama(
            model=model,
            base_url=kwargs.get("base_url", "http://localhost:11434"),
            temperature=kwargs.get("temperature", 0.7),
        )
    
    # ... 其他provider
```
