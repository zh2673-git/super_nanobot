# Local Deep Research 核心算法分析

> 本笔记深入分析项目中的核心算法，帮助理解支撑研究功能的底层技术。

---

## 第一步：点出核心算法的组件概念

- **[向量嵌入（Embeddings）]：把文字转换成"数字密码"的技术，让计算机能理解文字的"含义"，就像把一本小说浓缩成一串暗号**
- **[FAISS相似度搜索]：快速在海量"数字密码"中找到最相似的那个，就像在图书馆用索书号快速找到对应的书**
- **[搜索重排序（Reranking）]：对搜索结果进行"再筛选"，把最相关的结果排在前面，就像面试官先筛简历再通知面试**
- **[自适应速率限制（Adaptive Rate Limiting）]：智能控制搜索请求频率的算法，就像"红绿灯"根据车流自动调节信号**

---

**联系说明**：整个搜索和推荐的过程就像"图书馆找书"：
1. **你说出想找什么**（用户查询）
2. **图书管理员记下关键词**（向量嵌入）
3. **在系统中搜索**（FAISS搜索 + 搜索引擎API）

7. **把书给你**（返回结果）

## 第三步：详细解释核心算法

### 1. 向量嵌入（Embeddings）

**专业定义**：将文本转换为密集向量表示的算法，使得语义相似的文本在向量空间中距离较近。

**通俗解释与类比**：就像把"一句话"翻译成"一串密码"：
- 传统方式：关键词匹配（找包含"量子"的文档）
- 向量嵌入：找意思相近的文档（"量子"和"量子力学"意思相近，向量也相近）

**生活化例子**：
```
文字: "今天天气真好"
向量: [0.12, -0.34, 0.56, 0.89, -0.23, ...]  # 1024维向量

文字: "天气晴朗"
向量: [0.11, -0.33, 0.55, 0.90, -0.22, ...]  # 与上面很相似！

文字: "如何修电脑"
向量: [-0.78, 0.23, -0.45, 0.12, 0.67, ...]  # 完全不同！
```

**在此项目中的具体体现**：
```python
# 文件位置
src/local_deep_research/web_search_engines/engines/local_embedding_manager.py

class LocalEmbeddingManager:
    """本地嵌入管理器"""
    
    def __init__(
        self,
        embedding_model: str = "all-MiniLM-L6-v2",
        embedding_device: str = "cpu",
        embedding_model_type: str = "sentence_transformers",
    ):
        self.embedding_model = embedding_model
        self.embedding_device = embedding_device
        self.embedding_model_type = embedding_model_type
    
    def get_embedding(self, text: str) -> List[float]:
        """获取文本的向量表示"""
        embeddings = self.embeddings
        return embeddings.embed_query(text)
    
    def get_embeddings(self, texts: List[str]) -> List[List[float]]:
        """批量获取文本的向量表示"""
        embeddings = self.embeddings
        return embeddings.embed_documents(texts)
```

**支持的嵌入模型**：
| 模型 | 类型 | 维度 | 特点 |
|-----|------|------|------|
| all-MiniLM-L6-v2 | sentence_transformers | 384 | 轻量、快速 |
| all-mpnet-base-v2 | sentence_transformers | 768 | 精度更高 |
| Ollama本地模型 | ollama | 可配置 | 隐私优先 |

**性能分析**：
- **时间复杂度**：O(n × d)，n是文本数量，d是向量维度
- **空间复杂度**：O(n × d)，需要存储所有向量

### 2. FAISS相似度搜索

**专业定义**：Facebook AI Similarity Search，一个高效的向量相似度搜索库，可以在海量向量中快速找到最相似的k个向量。

**通俗解释与类比**：就像在图书馆用"索书号"快速找书：
- 没有索引：要从100万本书一本本翻（O(n)）
- 有FAISS索引：根据索书号直接定位（接近O(1)）

**生活化例子**：
```
场景：在100万篇论文中找与"量子计算"最相关的10篇

方法1: 暴力搜索
- 计算100万次相似度 → 慢！（可能需要几分钟）

方法2: FAISS索引
- 先建立索引（类似按字母排序的书架）
- 查找时只比较一小部分 → 快！（可能只需要几毫秒）
```

**在此项目中的具体体现**：
```python
# 文件位置
src/local_deep_research/web_search_engines/engines/search_engine_library.py

class SearchEngineLibrary:
    """知识库搜索引擎"""
    
    def __init__(self, collection_id: int):
        self.collection_id = collection_id
        self.rag_service = RAGService()
        self.vector_store = None
    
    def search(self, query: str, k: int = 10):
        """执行向量相似度搜索"""
        
        # 1. 获取查询的向量表示
        query_embedding = self.embedding_service.get_embedding(query)
        
        # 2. 使用FAISS进行相似度搜索
        docs_with_scores = self.vector_store.similarity_search_with_score(
            query=query,
            k=k,
        )
        
        # 3. 转换距离为相似度（距离越小越相似）
        results = []
        for doc, score in docs_with_scores:
            similarity = 1 / (1 + score)  # 距离转相似度
            results.append({
                "content": doc.page_content,
                "source": doc.metadata.get("source"),
                "similarity": similarity,
            })
        
        return results
```

**FAISS索引类型**：
| 索引类型 | 特点 | 适用场景 |
|---------|------|---------|
| IVF | 分组加速 | 数据量大时 |
| HNSW | 图索引 | 高精度场景 |
| Flat | 暴力精确 | 数据量小 |

**性能分析**：
- **搜索时间复杂度**：O(k × log(n))，n是向量数量
- **召回率**：使用HNSW时可达99%以上

### 3. 搜索重排序（Reranking）

**专业定义**：使用机器学习模型对初始搜索结果进行二次排序，提升结果相关性的算法。

**通俗解释与类比**：就像"面试筛选"：
- **初筛**（搜索）：根据关键词找出100份简历
- **精筛**（重排序）：仔细阅读后排出最合适的10份

**生活化例子**：
```
搜索词: "Python教程"

初筛结果（搜索引擎返回）:
1. Python官方网站 ⭐⭐⭐⭐⭐ (匹配多)
2. Python入门书籍 ⭐⭐⭐⭐ (匹配多)
3. 烤Python配方 ❌ (匹配"Python"这个词)
4. Python深度学习 ⭐⭐⭐ (匹配)
5. 爬虫Python ⭐⭐⭐ (匹配)

重排序后:
1. Python入门书籍 ⭐⭐⭐⭐ (最适合初学者)
2. Python官方网站 ⭐⭐⭐⭐⭐ (权威但难)
3. Python深度学习 ⭐⭐⭐ (适合进阶)
4. 爬虫Python ⭐⭐⭐ (实用技能)
5. Python食谱 ❌ (排序后靠后)
```

### 4. 自适应速率限制（Adaptive Rate Limiting）

**专业定义**：根据API返回的429错误（速率限制）动态调整请求频率的算法。

**通俗解释与类比**：就像"智能红绿灯"：
- 传统红绿灯：固定时间切换
- 自适应：检测车流，车多时绿灯时间长，车少时短

**生活化例子**：
```
场景：调用搜索引擎API

初始状态：
- 请求频率：每秒10次
- 被限制后：等待10秒

自适应调整后：
- 第一次被限制 → 降低到每秒5次
- 再次被限制 → 降低到每秒2次
- 连续成功 → 尝试提高到每秒3次
- 持续成功 → 提高到每秒4次
```

**在此项目中的具体体现**：
```python
# 文件位置
src/local_deep_research/web_search_engines/rate_limiting/tracker.py

class AdaptiveRateLimiter:
    """自适应速率限制器"""
    
    def __init__(self, initial_rate: int = 10):
        self.current_rate = initial_rate
        self.success_count = 0
        self.failure_count = 0
        self.attempt_history = []  # 记录每次尝试
    
    def can_proceed(self) -> bool:
        """判断是否可以继续请求"""
        return len(self.attempt_history) < self.current_rate
    
    def record_result(self, success: bool, wait_time: float = 0):
        """记录请求结果，用于动态调整"""
        # 记录这次尝试
        self.attempt_history.append({
            "success": success,
            "wait_time": wait_time,
            "timestamp": time.time()
        })
        
        # 如果被限制，增加失败计数
        if not success:
            self.failure_count += 1
            # 降低速率
            self.current_rate = max(1, self.current_rate // 2)
            # 添加等待时间
            time.sleep(wait_time)
        else:
            self.success_count += 1
            # 连续成功，逐步提高速率
            if self.success_count >= 5:
                self.current_rate = min(100, int(self.current_rate * 1.2))
        
        # 清理历史记录（保留最近100条）
        self.attempt_history = self.attempt_history[-100:]
    
    def get_optimal_wait_time(self) -> float:
        """计算最佳等待时间"""
        if not self.attempt_history:
            return 0
        
        # 分析过去失败的模式
        recent_failures = [a for a in self.attempt_history[-20:] if not a["success"]]
        if recent_failures:
            # 使用过去等待时间的平均值
            return sum(a["wait_time"] for a in recent_failures) / len(recent_failures)
        
        return 0
```

**算法特点**：
- **指数退避**：被限制后，按指数增长等待时间
- **渐进恢复**：连续成功后，逐步提高请求频率
- **智能学习**：根据历史数据预测最佳速率

## 算法性能对比

| 算法 | 时间复杂度 | 空间复杂度 | 精度 |
|-----|----------|----------|------|
| 向量嵌入 | O(n × d) | O(n × d) | N/A |
| FAISS搜索 | O(k × log(n)) | O(n × d) | ~99% |
| 重排序 | O(m × LLM) | O(m) | 高 |
| 自适应限速 | O(1) | O(h) | N/A |

**说明**：
- n = 文档数量
- d = 向量维度
- k = 返回结果数
- m = 搜索结果数
- h = 历史记录数
