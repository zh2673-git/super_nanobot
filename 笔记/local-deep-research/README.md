# Local Deep Research 项目概述

> 本笔记按照"理解坡度"学习法编写，旨在帮助零基础用户理解这个AI研究助手项目。

---

## 第一步：点出项目的核心组件概念

在深入了解这个项目之前，让我们先认识构成它的5个核心"积木块"：

- **[研究助手（Research Assistant）]：一个AI程序，能自动帮你查找资料、分析问题、生成报告，就像一个24小时工作的私人研究员**
- **[搜索引擎集成（Search Engine Integration）]：项目能同时调用20多种不同的搜索工具（Google、学术论文网站、Wikipedia等），就像一个"搜索万能钥匙"**
- **[大语言模型（LLM Provider）]：支持多种AI大脑（OpenAI、Claude、Ollama本地模型等），让用户可以根据需要选择最合适的"思考引擎"**
- **[知识库系统（Research Library）]：自动下载和整理资料的系统，像一个永不遗忘的"私人图书馆管理员"**
- **[加密数据库（Encrypted Database）]：用AES-256加密技术保护用户数据，就像给数据装上"防弹衣"，只有用户自己才能查看**

---

## 第二步：用图构建组件概念之间的联系

```
[用户提问]
    |
    v
[研究助手（核心大脑）]
    |-> 理解问题 -> 调用 [大语言模型] 分析需求
    |-> 制定策略 -> 选择合适的 [搜索引擎组合]
    |-> 执行搜索 -> 通过 [搜索引擎集成] 获取资料
    |-> 整理结果 -> 调用 [大语言模型] 总结分析
    |-> 保存资料 -> 通过 [知识库系统] 存入 [加密数据库]
    |
    v
[生成报告/答案]
```

**联系说明**：用户提出研究问题后，研究助手就像一个项目经理，先让"大脑"（大语言模型）理解问题，然后分配任务给不同的"搜索员"（搜索引擎），收集到的资料经过"大脑"处理后，一方面生成回答给用户，另一方面保存到"图书馆"（知识库系统）留作以后使用。所有数据都受到"防弹衣"（加密数据库）的保护。

## 第三步：详细解释

### 1. 研究助手（Research Assistant）

**专业定义**：一个AI驱动的自动化研究系统，能够执行深度迭代研究，使用多种大语言模型和搜索引擎来收集信息，并生成带有正确引用来源的研究报告。

**通俗解释与类比**：想象你需要了解"量子计算的最新发展"这个主题。传统方法是你自己上网搜索、阅读文章、总结要点，这可能需要几个小时。Local Deep Research就像一个超级高效的助手，你告诉它问题后，它会自动：
1. 制定研究计划（要查哪些方面）
2. 同时搜索多个来源（学术论文、新闻、技术博客）
3. 阅读和理解找到的内容
4. 提取关键信息
5. 生成结构化的报告

整个过程可能只需要几分钟，而且报告会有引用来源，方便你查证。

**在此项目中的具体体现**：
- 支持多种研究模式：快速摘要（30秒-3分钟）、详细研究（5-15分钟）、报告生成（10-30分钟）
- 提供20+种研究策略，用户可以根据需求选择
- 支持实时进度更新（通过WebSocket）
- 可以导出为PDF或Markdown格式

---

### 2. 搜索引擎集成（Search Engine Integration）

**专业定义**：一个模块化的搜索引擎框架，支持20+种不同的搜索来源，包括免费引擎（arXiv、PubMed、Wikipedia、SearXNG）和付费引擎（Google、Tavily、Brave Search）。

**通俗解释与类比**：就像一个"万能翻译器"，能够和世界上各种不同的"搜索引擎"对话。有的搜索引擎擅长找学术论文，有的擅长找新闻，有的擅长找技术文档。这个模块把这些能力都整合在一起，让用户可以用一个统一的接口访问所有这些搜索能力。

**在此项目中的具体体现**：

#### 免费搜索引擎
| 搜索引擎            | 用途   | 类比         |
| --------------- | ---- | ---------- |
| arXiv           | 学术论文 | 大学的学术资料库   |
| PubMed          | 医学文献 | 医学图书馆      |
| Wikipedia       | 百科知识 | 百科全书       |
| SearXNG         | 综合搜索 | 搜索引擎的"总调度" |
| GitHub          | 代码搜索 | 程序员的"代码仓库" |
| Wayback Machine | 历史网页 | 网络的"时光机"   |

#### 付费搜索引擎
| 搜索引擎 | 特点 | 类比 |
|---------|------|------|
| Tavily | AI优化的搜索 | 智能搜索助手 |
| Google | 综合搜索 | 搜索界的"全能选手" |
| Brave Search | 注重隐私 | 保护隐私的搜索 |

---

### 3. 大语言模型支持（LLM Provider）

**专业定义**：项目支持多种大语言模型 provider，包括本地模型（通过Ollama）和云端模型（OpenAI、Anthropic、Google等）。

**通俗解释与类比**：就像一个"多引擎汽车"，可以装不同的"发动机"：
- 本地模型（Ollama）= 装在车里的"国产发动机"，不用联网，速度快，但能力有限
- 云端模型（OpenAI等）= 租用"进口发动机"，能力强，但需要联网付费

用户可以根据自己的需求（隐私、速度、能力）选择合适的模型。

**在此项目中的具体体现**：

#### 本地模型（隐私优先）
- Llama 3
- Mistral
- Gemma
- DeepSeek

#### 云端模型（能力优先）
- OpenAI (GPT-4, GPT-3.5)
- Anthropic (Claude 3)
- Google (Gemini)
- 100+模型 via OpenRouter

---

### 4. 知识库系统（Research Library）

**专业定义**：一个自动化的资料管理系统，能够从网上下载学术论文、网页内容，并建立可搜索的向量索引。

**通俗解释与类比**：就像一个"私人图书馆"，不仅帮你买书（下载资料），还会给每本书做好"索引"（向量化存储），下次你想找相关内容时，只要描述一下，图书馆就能帮你找到最相关的资料。

**在此项目中的具体体现**：
- 自动从arXiv、PubMed等来源下载论文
- 提取文本并进行向量化处理
- 支持语义搜索（用自然语言描述就能找到相关内容）
- 支持LangChain集成，可以连接现有的向量数据库

---

### 5. 加密数据库（Encrypted Database）

**专业定义**：使用SQLCipher实现的AES-256加密数据库，为每个用户提供独立的加密存储空间。

**通俗解释与类比**：就像给每个用户分配一个"保险箱"。即使服务器被黑客攻破，没有密码也打不开这个保险箱。项目声称使用与Signal（一个著名的加密通讯应用）同级别的安全技术。

**重要安全说明**：
- 每个用户有独立的加密数据库
- 使用AES-256加密（目前最强级的民用加密之一）
- 没有"密码恢复"功能——如果忘记密码，数据无法恢复（这也是"零知识"安全的代价）
- 运行完全本地（使用Ollama + SearXNG）时，数据永远不会离开你的电脑

## 项目技术栈一览

| 类别 | 技术 | 用途 |
|-----|------|------|
| **Web框架** | Flask | 构建Web界面和API |
| **LLM框架** | LangChain | 连接各种AI模型 |
| **数据库** | SQLCipher | 加密存储 |
| **搜索** | SearXNG | 元搜索 |
| **本地AI** | Ollama | 运行本地模型 |
| **向量存储** | FAISS | 相似度搜索 |
| **前端** | Playwright | 自动化测试 |
