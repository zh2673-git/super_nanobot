# Local Deep Research 运行逻辑

> 本笔记分析项目如何执行一个完整的研究任务，帮助理解从用户提问到获得答案的整个流程。

---

## 第一步：点出运行流程的核心组件概念

- **[请求接收（Request Reception）]：Web服务器"守门员"，接收并初步处理用户的请求，就像电话接线员**
- **[任务队列（Task Queue）]：研究任务的"调度中心"，管理任务的排队和执行，就像餐厅的"叫号系统"**
- **[研究迭代器（Research Iterator）]：核心"工作引擎"，反复搜索和分析直到找到满意答案，就像餐厅的"主厨团队"**
- **[结果合成（Result Synthesis）]：AI"总结大师"，把收集到的信息整理成用户能看懂的报告，就像餐厅的"摆盘师"**
- **[WebSocket推送（Real-time Push）]：实时"直播员"，把研究进度实时展示给用户，就像餐厅里"叫号显示屏"**

---

## 第二步：用图构建运行流程的联系

```
[用户发起研究请求]
    |
    v
[请求接收：Flask路由]
    |-> 验证用户身份
    |-> 参数验证
    |-> 创建研究ID
    |
    v
[任务队列：放入后台处理]
    |-> 返回研究ID给前端
    |-> WebSocket开始"直播"
    |
    v
[研究迭代器：开始工作]
    |
    +--- 迭代1 --->
    |   |-> 生成搜索查询
    |   |-> 调用搜索引擎
    |   |-> 分析搜索结果
    |   |-> 决定是否需要更多搜索
    |   |
    |   +--- 迭代2 ---> (重复直到满足条件)
    |       ...
    |
    v
[结果合成：生成最终报告]
    |-> 让LLM总结发现
    |-> 添加引用来源
    |-> 格式化输出
    |
    v
[WebSocket推送结果]
    |
    v
[用户收到完整报告]
```

**联系说明**：整个流程就像去一家"智能餐厅"吃饭：
1. **接线员（请求接收）** 记录你的点餐（研究问题）
2. **叫号系统（任务队列）** 给你一个号码（研究ID），告诉你等会来取
3. **显示屏（WebSocket）** 实时显示"正在备菜中"、"正在烹饪中"
4. **主厨团队（研究迭代器）** 反复尝试不同的烹饪方法，直到味道满意
5. **摆盘师（结果合成）** 把做好的菜精美地摆好，加上装饰（引用来源）
6. **叫号取餐（推送结果）** 告诉你"您的餐好了"

## 第三步：详细解释运行流程

### 1. 请求接收（Request Reception）

**专业定义**：Flask路由接收用户的研究请求，进行身份验证和参数校验后，创建研究任务并返回任务ID。

**通俗解释与类比**：就像餐厅的"接线员"：
- 记录顾客的点餐（用户的研究问题）
- 确认顾客身份（登录验证）
- 给顾客一个号码牌（研究ID）
- 把订单传给厨房（放入任务队列）

**在此项目中的具体体现**：
```python
# 路由文件
src/local_deep_research/web/routes/research_routes.py

@bp.route("/api/start_research", methods=["POST"])
@login_required
def start_research():
    # 1. 获取并验证参数
    query = request.json.get("query")
    mode = request.json.get("mode", "quick")
    
    # 2. 创建研究ID
    research_id = str(uuid.uuid4())
    
    # 3. 放入任务队列（异步执行）
    run_research_process.delay(research_id, query, mode)
    
    # 4. 返回研究ID
    return jsonify({"research_id": research_id})
```

### 2. 任务队列（Task Queue）

**专业定义**：使用后台任务队列管理长时间运行的研究任务，确保Web请求可以快速返回，任务在后台异步执行。

**通俗解释与类比**：就像餐厅的"叫号系统"：
- 你点完餐后，不需要站在厨房等
- 系统给你一个号码，你可以坐着等
- 厨房做好了会"叫号"通知你

**在此项目中的具体体现**：
```python
# 使用Celery或类似的异步任务队列
# 任务定义
src/local_deep_research/web/services/research_service.py

def run_research_process(research_id, query, mode, **kwargs):
    """后台执行的研究任务"""
    # 获取研究引擎
    engine = get_research_engine(mode)
    
    # 执行研究
    result = engine.research(query)
    
    # 保存结果
    save_research_result(research_id, result)
    
    # 通过WebSocket推送结果
    notify_client(research_id, result)
```

### 3. 研究迭代器（Research Iterator）

**专业定义**：核心研究逻辑，使用循环迭代的方式执行搜索、分析、评估的循环，直到找到满意的答案或达到迭代上限。

**通俗解释与类比**：就像餐厅的"主厨团队"：
- 拿到点餐后，先"试做"一下（初步搜索）
- 尝尝味道如何（评估结果）
- 如果味道不够好，再调整一下（优化查询）
- 继续尝试，直到满意或者用完食材（达到迭代上限）

**在此项目中的具体体现**：
```python
# 研究迭代过程
async def research_iteration(query, max_iterations=5):
    results = []
    context = []
    
    for iteration in range(max_iterations):
        # 1. 根据当前上下文生成搜索查询
        search_query = generate_search_query(query, context)
        
        # 2. 执行搜索
        search_results = await search(search_query)
        
        # 3. 分析结果
        analysis = await analyze_results(search_results, context)
        
        # 4. 更新上下文
        context.append(analysis)
        results.append(search_results)
        
        # 5. 评估是否需要继续
        if should_stop(analysis):
            break
    
    return results, context
```

**研究模式**：
| 模式 | 迭代次数 | 用途 | 时间 |
|-----|---------|------|------|
| quick_summary | 1-2次 | 快速回答 | 30秒-3分钟 |
| detailed_research | 3-5次 | 详细分析 | 5-15分钟 |
| report_generation | 5-10次 | 完整报告 | 10-30分钟 |

### 4. 结果合成（Result Synthesis）

**专业定义**：使用大语言模型将收集到的搜索结果进行综合分析，生成结构化的研究报告，并添加引用来源。

**通俗解释与类比**：就像餐厅的"摆盘师"：
- 菜已经炒好了（信息收集完毕）
- 但需要精美摆盘（整理成可读的报告）
- 加上装饰和说明（添加引用来源）
- 让顾客不仅吃得舒服，还看得明白

**在此项目中的具体体现**：
```python
# 结果合成过程
async def synthesize_results(results, context):
    # 1. 准备prompt
    prompt = build_synthesis_prompt(results, context)
    
    # 2. 调用LLM生成报告
    llm = get_llm_provider()
    report = await llm.agenerate([prompt])
    
    # 3. 添加引用来源
    cited_report = add_citations(report, results)
    
    # 4. 格式化输出
    formatted = format_output(cited_report)
    
    return formatted

# 输出格式支持
- Markdown (.md)
- PDF (.pdf)
- JSON (.json)
```

## 错误处理与重试机制

### 1. 搜索失败重试
```python
# 搜索引擎失败时自动重试
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10))
async def search_with_retry(query, engine):
    return await engine.run(query)
```

### 2. LLM调用失败处理
```python
# LLM失败时的降级策略
try:
    result = await primary_llm.generate(prompt)
except Exception as primary_error:
    # 尝试备用LLM
    try:
        result = await fallback_llm.generate(prompt)
    except Exception as fallback_error:
        # 返回错误信息
        return ErrorResult(str(fallback_error))
```

### 3. 自适应速率限制
```python
# 智能重试系统，学习最佳等待时间
class AdaptiveRateLimiter:
    def __init__(self):
        self.wait_times = []
    
    def learn(self, success, wait_time):
        if success:
            self.wait_times.append(wait_time)
    
    def get_optimal_wait(self):
        return mean(self.wait_times) * 0.9  # 略微保守
```
