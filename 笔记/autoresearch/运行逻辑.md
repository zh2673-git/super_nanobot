# 运行逻辑

> 本笔记采用"理解坡度"学习法编写，详细分析Autoresearch项目的程序执行流程。

---

## 第一步：点出运行流程的核心组件概念

在深入理解程序如何运行之前，让我们先认识五个关键阶段：

- **[准备阶段]：环境初始化**（像是"开幕式"，准备好所有需要的工具和材料）
- **[模型构建阶段]：模型创建与初始化**（像是"搭建工厂"，把模型各部件组装起来）
- **[训练循环阶段]：固定时间预算训练**（像是"生产流水线"，反复迭代学习）
- **[评估阶段]：验证集评估**（像是"质检"，检查模型在未见过的数据上表现如何）
- **[结果输出阶段]：汇总报告**（像是"发成绩单"，输出最终结果）

---

## 第二步：用图构建运行流程的联系

```
[程序启动：uv run train.py]
    |
    v
[第一步：环境初始化]
    |-> 设置随机种子(42)
    |-> 设置CUDA设备
    |-> 配置AMP自动混合精度
    |-> 加载Tokenizer
    |
    v
[第二步：构建模型配置]
    |-> 计算模型维度(DEPTH * ASPECT_RATIO)
    |-> 计算注意力头数
    |-> 创建GPTConfig对象
    |
    v
[第三步：创建并初始化模型]
    |-> 在meta设备创建模型(不占内存)
    |-> 移动到CUDA设备
    |-> 初始化权重(init_weights)
    |
    v
[第四步：设置优化器]
    |-> 为不同参数设置不同学习率
    |-> 创建MuonAdamW优化器
    |-> 编译模型(torch.compile)
    |
    v
[第五步：创建数据加载器]
    |-> 预取第一批数据
    |
    v
[第六步：训练循环(while True)]
    |-> 梯度累积循环
    |   |-> 前向传播计算loss
    |   |-> 反向传播计算梯度
    |   |-> 预取下一批数据
    |   |
    |-> 更新学习率/动量/权重衰减
    |-> 执行优化器step
    |-> 清空梯度
    |-> 记录训练loss
    |-> 检查是否超时
    |   |-> 是 -> 跳出循环
    |   |-> 否 -> 继续下一轮
    |
    v
[第七步：最终评估]
    |-> 切换到eval模式
    |-> 在验证集上计算val_bpb
    |
    v
[第八步：输出结果汇总]
    |-> 打印val_bpb
    |-> 打印训练时间、内存使用等
    |
    v
[程序结束]
```

**联系说明**：整个程序就像一条"自动化生产线"。首先准备好所有工具（初始化），然后搭建工厂（模型构建），然后启动流水线开始生产（训练循环），每生产一段时间就检查一下产品质量（评估），最后汇总所有数据输出报告。核心特点是：这条流水线只运行5分钟，时间一到就停工，无论生产了多少产品。

## 第三步：运行流程详细解释

### 一、环境初始化阶段

#### 1.1 PyTorch和CUDA设置

**专业定义**：设置随机种子、CUDA设备、混合精度等基础环境。

**通俗解释与类比**：这就像是在开始工作前做好一切准备工作：
- **随机种子**：把随机数生成器"锁死"，让每次运行结果可复现——像是每次考试都用同一套试卷
- **CUDA设备**：指定用GPU来计算——像是指定去哪个车间工作
- **混合精度**：用bf16计算，牺牲一点精度换取速度——像是用估算而不是精确计算，能快很多

**在此项目中的具体体现**：
```python
torch.manual_seed(42)
torch.cuda.manual_seed(42)
torch.set_float32_matmul_precision("high")
device = torch.device("cuda")
autocast_ctx = torch.amp.autocast(device_type="cuda", dtype=torch.bfloat16)
```

### 二、模型配置构建阶段

#### 2.1 配置参数计算

**专业定义**：根据用户定义的DEPTH、ASPECT_RATIO、HEAD_DIM等参数，自动计算模型的具体配置。

**通俗解释与类比**：这就像是根据"设计图纸"来"计算材料用量"：
- DEPTH = 8层 → 确定需要多少个"处理车间"
- ASPECT_RATIO = 64 → 确定车间的"宽度"
- HEAD_DIM = 128 → 确定每个"审核员"的专业程度
- 最终计算出：n_embd = 8 * 64 = 768，n_head = 768 / 128 = 6

**在此项目中的具体体现**：
```python
ASPECT_RATIO = 64
HEAD_DIM = 128
DEPTH = 8

def build_model_config(depth):
    base_dim = depth * ASPECT_RATIO  # 8 * 64 = 512
    model_dim = ((base_dim + HEAD_DIM - 1) // HEAD_DIM) * HEAD_DIM  # 768
    num_heads = model_dim // HEAD_DIM  # 6
    return GPTConfig(
        sequence_len=MAX_SEQ_LEN, vocab_size=vocab_size,
        n_layer=depth, n_head=num_heads, n_kv_head=num_heads, n_embd=model_dim,
    )
```

### 三、模型创建与初始化阶段

#### 3.1 模型创建

**专业定义**：使用torch.device("meta")创建模型结构，然后移动到实际设备并初始化权重。

**通俗解释与类比**：这就像是在"虚拟车间"先画出图纸，确认没问题后再"建造真正的工厂"：
- **meta设备**：不占用实际内存，只是创建模型结构——像是先画设计图
- **to_empty**：把设计图变成真正的工厂——移动到GPU
- **init_weights**：给工厂安装机器和设备——初始化权重

**在此项目中的具体体现**：
```python
with torch.device("meta"):
    model = GPT(config)  # 创建模型结构
model.to_empty(device=device)  # 移动到GPU
model.init_weights()  # 初始化权重
```

#### 3.2 权重初始化

**专业定义**：使用特定的初始化策略初始化模型各部分权重。

**通俗解释与类比**：这就像是在安装机器时调整"灵敏度"：
- **Embedding层**：使用较大的标准差(1.0)——像是让接收器"敏感"一些
- **LM Head**：使用极小的标准差(0.001)——像是让输出"保守"一些
- **Q/K/V投影**：使用缩放后的均匀分布——像是平衡"探索"和"保守"
- **MLP**：类似Q/K/V的处理方式
- **残差缩放**：resid_lambdas设为1.0，x0_lambdas设为0.1

**在此项目中的具体体现**：
```python
def init_weights(self):
    torch.nn.init.normal_(self.transformer.wte.weight, mean=0.0, std=1.0)
    torch.nn.init.normal_(self.lm_head.weight, mean=0.0, std=0.001)
    # 投影层使用均匀分布
    s = 3**0.5 * n_embd**-0.5
    for block in self.transformer.h:
        torch.nn.init.uniform_(block.attn.c_q.weight, -s, s)
        # ... 其他投影层类似
```

### 四、优化器设置阶段

#### 4.1 混合优化器配置

**专业定义**：为不同类型的参数设置不同的优化器和学习率。

**通俗解释与类比**：这就像是为不同"岗位"配备不同的"培训方式"：
- **LM Head（unembedding）**：使用AdamW，学习率较低(0.004)——像是"老员工"，不需要大学习率
- **Embedding层**：使用AdamW，学习率较高(0.6)——像是"新人"，需要大学习率快速学习
- **矩阵参数（Q/K/V/MLP等）**：使用Muon，学习率中等(0.04)——这是核心员工，使用特殊的"正交优化器"
- **标量参数（resid_lambdas等）**：使用AdamW，学习率很高(0.5)——像是"调节旋钮"，需要快速调整

**在此项目中的具体体现**：
```python
param_groups = [
    dict(kind='adamw', params=lm_head_params, lr=UNEMBEDDING_LR * dmodel_lr_scale),
    dict(kind='adamw', params=embedding_params, lr=EMBEDDING_LR * dmodel_lr_scale),
    dict(kind='adamw', params=value_embeds_params, lr=EMBEDDING_LR * dmodel_lr_scale),
    dict(kind='adamw', params=resid_params, lr=scalar_lr * 0.01),
    dict(kind='adamw', params=x0_params, lr=scalar_lr),
]
# 矩阵参数使用Muon
for shape in sorted({p.shape for p in matrix_params}):
    param_groups.append(dict(kind='muon', params=group_params, lr=MATRIX_LR, ...))
```

#### 4.2 学习率缩放

**专业定义**：根据模型维度缩放学习率，遵循1/sqrt(d_model)规律。

**通俗解释与类比**：这就像是根据"工厂规模"调整"培训强度"：
- 更大的模型（d_model更大）需要更小的学习率
- 缩放因子 = (d_model / 768)^(-0.5)
- 默认配置下，d_model=768，缩放因子=1.0

**在此项目中的具体体现**：
```python
dmodel_lr_scale = (model_dim / 768) ** -0.5
```

### 五、训练循环阶段

#### 5.1 梯度累积

**专业定义**：通过多次前向/反向传播累积梯度，实现更大的有效批次大小。

**通俗解释与类比**：这就像是把"大任务"拆成"小任务"来完成：
- **目标批次**：TOTAL_BATCH_SIZE = 2^19 ≈ 52万tokens
- **实际批次**：DEVICE_BATCH_SIZE * MAX_SEQ_LEN = 128 * 2048 ≈ 26万tokens
- **累积步数**：grad_accum_steps = 2
- 也就是说，每次需要做2次前向传播才更新一次参数

**在此项目中的具体体现**：
```python
tokens_per_fwdbwd = DEVICE_BATCH_SIZE * MAX_SEQ_LEN
grad_accum_steps = TOTAL_BATCH_SIZE // tokens_per_fwdbwd

for micro_step in range(grad_accum_steps):
    with autocast_ctx:
        loss = model(x, y)
    loss = loss / grad_accum_steps
    loss.backward()
    x, y, epoch = next(train_loader)
```

#### 5.2 学习率调度

**专业定义**：根据训练进度动态调整学习率，包含warmup和warmdown阶段。

**通俗解释与类比**：这就像是一个"温度调节器"：
- **Warmup阶段**：开始时学习率从0逐渐上升到最大值——像是"热身"
- **平稳阶段**：保持最高学习率——像是"全速前进"
- **Warmdown阶段**：学习率从最大值逐渐下降到最终值——像是"减速停车"

**在此项目中的具体体现**：
```python
WARMUP_RATIO = 0.0      # 默认不warmup
WARMDOWN_RATIO = 0.5    # 一半时间用于warmdown
FINAL_LR_FRAC = 0.0     # 最终学习率为0

def get_lr_multiplier(progress):
    if progress < WARMUP_RATIO:
        return progress / WARMUP_RATIO if WARMUP_RATIO > 0 else 1.0
    elif progress < 1.0 - WARMDOWN_RATIO:
        return 1.0
    else:
        cooldown = (1.0 - progress) / WARMDOWN_RATIO
        return cooldown * 1.0 + (1 - cooldown) * FINAL_LR_FRAC
```

#### 5.3 Muon优化器动态调整

**专业定义**：Muon优化器的动量和权重衰减随训练进度变化。

**通俗解释与类比**：这像是调节"跑步节奏"：
- **动量**：从0.85逐渐增加到0.95——开始时小步快跑，后来大步流星
- **权重衰减**：从初始值逐渐降到0——开始时严格控制，后期放宽

**在此项目中的具体体现**：
```python
def get_muon_momentum(step):
    frac = min(step / 300, 1)
    return (1 - frac) * 0.85 + frac * 0.95

def get_weight_decay(progress):
    return WEIGHT_DECAY * (1 - progress)
```

#### 5.4 训练监控与早停

**专业定义**：监控训练过程中的损失值，检测异常情况并提前终止。

**通俗解释与类比**：这像是安装了"安全阀"：
- **NaN检测**：如果损失变成NaN，立即停止——像是发现机器故障就停机
- **爆炸检测**：如果损失超过100，立即停止——像是压力过大就泄压
- **时间预算**：固定5分钟后停止——像是定了闹钟，时间到就收工

**在此项目中的具体体现**：
```python
# 检查异常
if math.isnan(train_loss_f) or train_loss_f > 100:
    print("FAIL")
    exit(1)

# 检查时间预算
if step > 10 and total_training_time >= TIME_BUDGET:
    break
```

### 六、评估阶段

#### 6.1 验证集评估

**专业定义**：在验证集上计算val_bpb（验证比特每字节）指标。

**通俗解释与类比**：这像是"期末考试"：
- 用模型在验证集上"做题"
- 计算每个字节的平均比特数（bpb）
- 这是最终衡量模型好坏的标准——分数越低越好

**在此项目中的具体体现**：
```python
model.eval()
with autocast_ctx:
    val_bpb = evaluate_bpb(model, tokenizer, DEVICE_BATCH_SIZE)
```

### 七、结果输出阶段

#### 7.1 最终统计信息

**专业定义**：输出完整的训练统计信息。

**通俗解释与类比**：这像是"发放成绩单"：
- **val_bpb**：最终成绩
- **training_seconds**：有效训练时间（不含编译等开销）
- **total_seconds**：总运行时间（含启动开销）
- **peak_vram_mb**：内存峰值
- **mfu_percent**：模型FLOPs利用率
- **total_tokens_M**：训练的总token数
- **num_steps**：更新的步数
- **num_params_M**：模型参数量

**在此项目中的具体体现**：
```python
print(f"val_bpb:          {val_bpb:.6f}")
print(f"training_seconds: {total_training_time:.1f}")
print(f"total_seconds:    {t_end - t_start:.1f}")
print(f"peak_vram_mb:     {peak_vram_mb:.1f}")
print(f"mfu_percent:      {steady_state_mfu:.2f}")
print(f"total_tokens_M:   {total_tokens / 1e6:.1f}")
print(f"num_steps:        {step}")
print(f"num_params_M:     {num_params / 1e6:.1f}")
print(f"depth:            {DEPTH}")
```
