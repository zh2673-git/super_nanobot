# Autoresearch 项目笔记

> 本笔记采用"理解坡度"学习法编写，帮助零基础读者从简单到复杂逐步理解这个自主AI研究项目。

---

## 第一步：点出项目的核心组件概念

在深入了解这个项目之前，让我们先认识它的五个核心"积木块"：

- **[项目定位]：自主LLM训练研究系统**（想象成一个"AI科学实验室"，让AI自己当科学家做实验）
- **[核心文件-train.py]：训练脚本**（这是AI代理唯一可以修改的"实验笔记本"，记录了模型如何学习和成长）
- **[核心文件-prepare.py]：数据准备工具**（像是"图书馆管理员"，负责获取训练数据和准备Tokenizer）
- **[核心文件-program.md]：代理指令**（像是"实验指导手册"，告诉AI代理应该如何开展实验）
- **[评估指标-val_bpb]：验证比特每字节**（像是"成绩单"，衡量模型表现好坏，数值越低越好）

```
    |-> 读取README.md（了解项目背景）
```

## 第三步：项目详细解释

### 一、项目定位与背景

**专业定义**：Autoresearch是一个自主预训练研究系统，旨在让AI代理能够自主进行LLM（大型语言模型）的训练实验。

**通俗解释与类比**：想象一下，你是一名科学家，需要反复做实验来改进一个实验结果。通常，你需要手动设计实验、运行、记录、分析，这个过程既繁琐又耗时。Autoresearch就像是把这一切自动化了——你给AI一个目标（比如让模型表现更好），AI会自动设计方案、做实验、评估结果，如果结果变好就保留，不好就放弃，然后继续尝试新方案。你只需要第二天早上醒来查看实验日志就行了。

**在此项目中的具体体现**：
- 项目来源于Andrej Karpathy的设想，他在2026年3月的推特中描述了这个"AI自主研究"的愿景
- 核心理念：给AI一个小型但真实的LLM训练环境，让它能够整夜自主实验
- 训练运行固定的5分钟时间预算（无论硬件如何），评估指标是val_bpb（验证比特每字节），越低越好

### 二、核心文件与职责

#### 2.1 prepare.py - 数据准备工具

**专业定义**：负责一次性数据准备的Python脚本，包括下载训练数据、训练BPE tokenizer、提供运行时工具函数（dataloader、评估函数）。

**通俗解释与类比**：prepare.py就像是"后勤部门"。在真正开始训练之前，需要：
1. 去HuggingFace下载数据集（climbmix-400b-shuffle）
2. 训练一个Tokenizer（把文字转换成数字ID的编码器）
3. 准备好数据加载器（按批次提供训练数据）
4. 准备好评估函数（计算val_bpb）

这些工作只需要做一次，之后就可以反复使用。这个文件被设计为"不可修改"的，确保所有实验都在相同的数据基础上进行，保证实验的可比性。

**在此项目中的具体体现**：
- MAX_SEQ_LEN = 2048：上下文长度
- TIME_BUDGET = 300：5分钟训练时间预算
- 数据存储在 ~/.cache/autoresearch/
- 使用GPT-4风格的BPE分词器，词汇量8192

#### 2.2 train.py - 训练脚本

**专业定义**：包含完整GPT模型定义、优化器（Muon + AdamW）、训练循环的单文件脚本，是AI代理唯一可以修改的文件。

**通俗解释与类比**：train.py是整个项目的"核心实验记录本"。它包含了：
- **模型架构**：一个GPT风格的Transformer模型
- **优化器**：Muon（用于2D矩阵参数）+ AdamW（用于其他参数）
- **训练循环**：固定5分钟运行，然后评估val_bpb

这是AI代理的" playground"（游乐场），代理可以对模型架构、超参数、训练策略等进行任何修改，然后运行实验看效果。

**在此项目中的具体体现**：
- 默认配置：DEPTH=8层，ASPECT_RATIO=64，HEAD_DIM=128
- 默认窗口模式："SSSL"（交替使用短窗口和长窗口注意力）
- 使用Flash Attention 3加速

#### 2.3 program.md - 代理指令

**专业定义**：为AI代理提供的实验指导文件，包含实验设置、实验规则、输出格式、实验循环等完整指令。

**通俗解释与类比**：program.md就像是导师给研究生的"实验手册"。它详细说明了：
1. 如何开始一个实验（创建分支、验证数据）
2. 可以做什么、不可以做什么
3. 如何记录结果
4. 实验循环的完整流程

代理只需要按照这个手册执行，就能自主完成整个研究过程，不需要人类干预。

**在此项目中的具体体现**：
- 实验周期：约5分钟/次，约12次/小时
- 结果记录：保存到results.tsv（不提交到git）
- 决策规则：val_bpb降低则保留，否则丢弃

### 三、项目技术栈

**专业定义**：项目使用的主要技术组件和依赖。

**通俗解释与类比**：就像一个餐厅需要厨房设备一样，这个项目需要一系列"工具"来工作：
- **Python**：编程语言
- **PyTorch**：深度学习框架（像是"炒菜锅"）
- **Flash Attention 3**：高效注意力计算库（像是"高压锅"，让运算更快）
- **uv**：Python包管理工具（像是"采购员"，负责安装依赖）
- **rustbpe**和**tiktoken**：Tokenizer实现（像是"翻译员"，把文字转成数字）

### 四、实验流程详解

**专业定义**：从启动到完成一轮实验的完整流程。

**通俗解释与类比**：整个实验流程就像是一个"自动化生产流水线"：

1. **准备阶段**：创建实验分支（如autoresearch/mar5）
2. **第一轮**：运行baseline（不做任何修改），记录初始val_bpb
3. **修改train.py**：AI想出一个改进主意，修改代码
4. **提交代码**：git commit保存这次修改
5. **运行实验**：执行`uv run train.py`，训练5分钟
6. **评估结果**：查看val_bpb值
7. **决策**：如果val_bpb降低了（变好），保留这次修改；如果没降低或变差，丢弃（git reset）
8. **循环**：回到步骤3，继续下一轮实验

**在这个项目中的具体体现**：
- 每轮实验约5分钟
- 人类可以整晚不在，AI会一直运行实验
- 早上醒来查看results.tsv可以看到一夜的实验结果

### 五、快速开始指南

**步骤一：安装依赖**
```bash
# 安装uv包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 安装项目依赖
uv sync
```

**步骤二：准备数据**
```bash
# 下载数据并训练Tokenizer（约2分钟）
uv run prepare.py
```

**步骤三：运行单次训练实验（5分钟）**
```bash
uv run train.py
```

**步骤四：启动自主实验模式**
