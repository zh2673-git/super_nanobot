### 1.2 FastAPI 应用初始化

**位置**：`src/sirchmunk/api/main.py`

**详细步骤**：

```python
# 1. 加载环境变量
_work_path = Path(os.getenv("SIRCHMUNK_WORK_PATH", "~/.sirchmunk"))
_env_file = _work_path / ".env"
load_dotenv(_env_file)  # 加载 .env 文件

# 2. 创建 FastAPI 应用
app = FastAPI(
    title="Sirchmunk API",
    description="APIs for Sirchmunk",
    version="1.0.0"
)

# 3. 配置 CORS 中间件
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# 4. 注册所有路由
app.include_router(knowledge_router)     # 知识管理
app.include_router(settings_router)      # 设置
app.include_router(history_router)        # 历史记录
app.include_router(chat_router)          # 聊天
app.include_router(monitor_router)       # 监控
app.include_router(search_router)        # 搜索

# 5. 如果启用 Web UI，挂载静态文件
if _serve_ui:
    app.mount("/", StaticFiles(...))
```

### 2.2 闲聊查询检测

**目的**：对于简单的问候语，直接返回闲聊响应，避免不必要的 LLM 调用

**实现代码位置**：`search.py` 中的正则表达式匹配

```python
# 闲聊查询正则表达式
_CHAT_QUERY_RE = re.compile(
    r"^("
    # 问候语 (中/英/日/韩)
    r"你好|您好|嗨|哈喽|hello|hi|hey"
    r"|good\s*morning|good\s*afternoon"
    # 身份问题
    r"|who\s+are\s+you|你是谁"
    r"|what\s+can\s+you\s+do|你能做什么"
    # 感谢/再见
    r"|thank\s*you|thanks|谢谢"
    r"|bye|goodbye|再见"
    # 测试
    r"|test|ping"
    r")[\s!?。.]*$",
    re.IGNORECASE,
)
```

**工作流程**：
```
收到查询
    │
    ▼
正则匹配 _CHAT_QUERY_RE
    │
    ├──► 匹配成功 → 返回闲聊响应
    │
    └──► 匹配失败 → 进入搜索流程
```

### 2.3 关键词提取

**目的**：从自然语言查询中提取关键词，提升检索效果

**实现**：
1. 调用 LLM 进行关键词提取
2. 使用预设模板 `generate_keyword_extraction_prompt`

**流程**：
```
原始查询: "如何实现 Python 异步编程？"
    │
    ▼
LLM 关键词提取
    │
    ▼
提取结果: ["Python", "异步编程", "async", "asyncio"]
    │
    ▼
用于后续检索
```

### 2.4 文本检索 (GrepRetriever)

**核心组件**：`retrieve/text_retriever.py`

**检索方式**：使用 `ripgrep-all` (rga) 进行全文检索

**检索流程**：
```
关键词列表
    │
    ▼
构造 rga 命令
    │
    ▼
执行 shell 命令
    │
    ▼
解析输出结果
    │
    ▼
返回匹配片段列表
```

**示例命令**：
```bash
rga -n --json "Python" /path/to/search
```

### 2.5 证据处理

**目的**：对检索结果进行排序、过滤和去重

**处理步骤**：
1. **相关性排序**：根据关键词匹配程度排序
2. **去重**：移除重复的证据
3. **过滤**：过滤掉低质量或不相关的证据
4. **截断**：根据上下文窗口限制长度


## 五、代理执行流程

### 5.1 ReAct Agent 工作流程

**位置**：`src/sirchmunk/agentic/react_agent.py`

```
用户复杂查询
    │
    ▼
ReAct Agent 执行循环
    │
    ┌────────────────────────────────────┐
    │  1. Thought (思考)                 │
    │     分析当前状态，决定下一步行动    │
    └─────────────┬──────────────────────┘
                  │
                  ▼
    ┌────────────────────────────────────┐
    │  2. Action (行动)                  │
    │     选择并执行工具                 │
    └─────────────┬──────────────────────┘
                  │
                  ▼
    ┌────────────────────────────────────┐
    │  3. Observation (观察)             │
    │     获取工具执行结果               │
    └─────────────┬──────────────────────┘
                  │
                  ▼
         [判断是否完成]
           │        │
          完成    继续
           │        │
           └────┬───┘
                ▼
        返回最终结果
```

### 5.2 工具选择逻辑

ReAct Agent 可以使用的工具：

| 工具 | 功能 | 适用场景 |
|------|------|----------|
| **search** | 执行搜索 | 需要查找信息时 |
| **scan_directory** | 扫描目录 | 需要了解文件结构时 |
| **query_knowledge** | 查询知识库 | 需要获取已有知识时 |
