## 一、蒙特卡洛证据采样算法

### 1.1 算法概述

**算法名称**：Monte Carlo Evidence Sampling (蒙特卡洛证据采样)

**应用场景**：在搜索结果中对证据进行采样和排序，选择最相关的证据供 LLM 使用

**核心思想**：借鉴蒙特卡洛方法的思想，通过随机采样和迭代优化，找到最优的证据组合

**类比解释**：就像一个**选书委员**要从图书馆海量藏书中挑选出最适合回答用户问题的书籍：
- 书籍太多，不可能全部阅读
- 需要用智能的方法筛选出最相关的部分
- 每次筛选都要考虑多样性和相关性

### 1.2 算法原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                  蒙特卡洛证据采样流程                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  输入：候选证据列表 (Evidence Units)                                 │
│                                                                     │
│  1. 初始化                                                          │
│     ├── 随机选择初始证据集                                          │
│     └── 设置采样参数                                                │
│                                                                     │
│  2. 迭代采样 (多次循环)                                            │
│     ├── 随机选择一个候选证据                                        │
│     ├── 计算加入该证据后的得分                                      │
│     ├── 决定是否接受该证据                                         │
│     └── 更新证据集                                                  │
│                                                                     │
│  3. 输出最优证据集                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.3 算法实现

```python
def monte_carlo_sampling(
    candidates: List[EvidenceUnit],
    k: int = 10,
    iterations: int = 100
) -> List[EvidenceUnit]:
    """
    蒙特卡洛证据采样
    
    参数：
    - candidates: 候选证据列表
    - k: 需要选取的证据数量
    - iterations: 迭代次数
    
    返回：
    - 选中的证据列表
    """
    selected = []
    
    for _ in range(iterations):
        # 随机选择一个候选
        candidate = random.choice(candidates)
        
        # 计算得分（考虑相关性和多样性）
        score = calculate_score(candidate, selected, candidates)
        
        # 如果得分足够高，加入选中列表
        if score > threshold:
            selected.append(candidate)
        
        # 达到数量限制
        if len(selected) >= k:
            break
    
    return selected[:k]
```

### 1.4 性能分析

| 指标 | 说明 | 优化建议 |
|------|------|----------|
| **时间复杂度** | O(n * k * iterations) | 减少迭代次数或提前终止 |
| **空间复杂度** | O(k) | 只需要保存选中结果 |
| **采样质量** | 随机性保证多样性 | 可调节 temperature 参数 |

## 二、ReAct 推理算法

### 2.1 算法概述

**算法名称**：ReAct (Reasoning + Acting)

**应用场景**：处理复杂的多步骤搜索和推理任务

**核心思想**：交替进行"思考"和"行动"，通过观察行动结果来指导下一步推理

**类比解释**：就像一个**侦探破案**：
- 侦探不会一次性得出结论
- 每一步都会思考："我现在知道什么？"
- 然后采取行动："我去现场看看"
- 观察结果后继续推理

### 2.2 算法原理

```
┌─────────────────────────────────────────────────────────────────────┐
│                       ReAct 执行循环                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌──────────────┐                                                  │
│  │   初始状态   │                                                  │
│  └──────┬───────┘                                                  │
│         │                                                          │
│         ▼                                                          │
│  ┌──────────────────────────────────────────┐                     │
│  │  Thought: 思考                           │                     │
│  │  "根据目前的信息，我需要..."             │                     │
│  └──────────────┬───────────────────────────┘                     │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                     │
│  │  Action: 行动                           │                     │
│  │  调用工具: search, scan, query          │                     │
│  └──────────────┬───────────────────────────┘                     │
│                 │                                                   │
│                 ▼                                                   │
│  ┌──────────────────────────────────────────┐                     │
│  │  Observation: 观察结果                    │                     │
│  │  工具返回的结果                          │                     │
│  └──────────────┬───────────────────────────┘                     │
│                 │                                                   │
│                 ▼                                                   │
│         [判断是否完成?]                                            │
│           │        │                                               │
│          是       否                                               │
│           │        │                                               │
│           ▼        │                                               │
│  ┌──────────────┐ │                                               │
│  │   返回结果   │ │                                               │
│  └──────────────┘ │                                               │
│                    │                                               │
│         ◄─────────┘                                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 算法实现

```python
async def react_agent(
    query: str,
    tools: List[Tool],
    max_iterations: int = 10
) -> str:
    """ReAct Agent 实现"""
    
    # 初始化上下文
    context = {
        "query": query,
        "thoughts": [],
        "actions": [],
        "observations": []
    }
    
    for iteration in range(max_iterations):
        # 1. Thought: 思考
        thought = await llm.think(context)
        context["thoughts"].append(thought)
        
        # 检查是否完成
        if is_complete(thought):
            return format_answer(context)
        
        # 2. Action: 选择并执行工具
        action = await llm.choose_action(thought, tools)
        context["actions"].append(action)
        
        # 执行工具
        observation = await execute_tool(action)
        context["observations"].append(observation)
        
        # 3. Observation: 观察结果
        # 将结果加入上下文，用于下一步推理
        context["query"] += f"\n观察: {observation}"
    
    # 超过最大迭代次数
    return format_answer(context)
```

|------|----------|----------|

| **scan_directory** | 需要了解结构 | 涉及文件列表 |

## 三、文本检索算法

### 3.1 基于 ripgrep-all 的全文检索

**算法名称**：GrepRetriever

**应用场景**：在大规模文件系统中快速检索文本内容

**核心思想**：利用 ripgrep-all (rga) 的高效正则表达式匹配能力

**类比解释**：就像一个**超级搜索器**：
- 不是逐个文件打开查看
- 而是使用专业的搜索工具
- 一次性在所有文件中找到匹配内容

### 3.2 检索流程

```
用户查询
    │
    ▼
关键词预处理
    │
    ▼
构造 rga 命令
    │
    ├──► rga -n --json "关键词" 路径
    │
    ▼
执行 shell 命令
    │
    ▼
解析 JSON 输出
    │
    ├──► 提取文件路径
    ├──► 提取行号
    └──► 提取匹配内容
    │
    ▼
结果排序和过滤
    │
    ▼
返回结果列表
```

### 3.3 rga 命令示例

```bash
# 基本搜索
rga "Python" /path/to/search

# 带行号
rga -n "Python" /path/to/search

# JSON 输出
rga --json "Python" /path/to/search

# 正则表达式
rga -e "async.*await" /path/to/search

# 忽略大小写
rga -i "python" /path/to/search
```

    │
    ▼
LLM 关键词提取
    │
    ├──► 系统提示词

## 五、知识聚类算法

### 5.1 算法概述

**算法名称**：Knowledge Clustering

**应用场景**：将相关知识组织成知识集群

**核心思想**：基于内容相似性将证据单元分组

### 5.2 聚类流程

```
证据单元列表
    │
    ▼
计算相似度矩阵
    │
    ▼
层次聚类 (Hierarchical Clustering)
    │
    ├──► 计算簇间距离
    ├──► 合并相似簇
    └──► 迭代直到满足条件
    │
    ▼
生成知识集群
    │
    ├──► 簇1: [证据1, 证据2, 证据5]
    ├──► 簇2: [证据3, 证据4]
    └──► 簇3: [证据6, 证据7, 证据8]
```
