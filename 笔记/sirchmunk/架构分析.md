## 二、核心组件

### 2.1 组件概览

| 组件 | 位置 | 功能描述 | 类比 |
|------|------|----------|------|
| **AgenticSearch** | `search.py` | 智能搜索核心，协调整个搜索流程 | 公司的总经理，负责统筹安排 |
| **DocQA** | `doc_qa.py` | 文档问答，处理自然语言问题 | 公司的客服主管，理解问题并回答 |
| **KnowledgeBase** | `learnings/knowledge_base.py` | 知识库管理，维护知识集群 | 公司的图书馆管理员 |
| **GrepRetriever** | `retrieve/text_retriever.py` | 文本检索，使用 rga 进行搜索 | 公司的搜索专家 |
| **DuckDBStorage** | `storage/duckdb.py` | 数据持久化 | 公司的档案室 |
| **ReActAgent** | `agentic/react_agent.py` | 推理行动代理 | 公司的执行经理 |

### 2.2 各组件详细说明

#### 2.2.1 搜索模块 (AgenticSearch)

**核心职责**：作为搜索流程的总协调者，AgenticSearch 负责：
- 接收用户查询
- 分析查询意图
- 调度各种工具完成搜索
- 整合结果并返回

**类比解释**：就像一个餐厅的**店长**：
1. 顾客进来点餐（用户发起搜索）
2. 店长分析顾客想要什么（分析查询意图）
3. 店长安排厨师、服务员各司其职（调度工具）
4. 最终把做好的菜端给顾客（返回结果）

**位置**：`src/sirchmunk/search.py`

#### 2.2.2 文档问答模块 (DocQA)

**核心职责**：
- 接收自然语言问题
- 从知识库中检索相关信息
- 使用 LLM 生成答案
- 返回结构化结果

**类比解释**：就像一个**问答主持人**：
- 听众提问 → 用户问题
- 主持人思考需要找什么资料 → 检索知识
- 主持人组织语言回答 → LLM 生成答案

**位置**：`src/sirchmunk/doc_qa.py`

#### 2.2.3 知识库模块 (KnowledgeBase)

**核心职责**：
- 管理知识集群（Knowledge Clusters）
- 处理证据单元（Evidence Units）
- 维护知识生命周期

**类比解释**：就像一个**图书馆系统**：
- 知识集群 → 书籍分类（如"计算机"类）
- 证据单元 → 书中的章节或段落
- 生命周期 → 书籍从采购到借阅的状态

**位置**：`src/sirchmunk/learnings/knowledge_base.py`

## 三、设计模式

### 3.1 分层架构

Sirchmunk 采用经典的分层架构：

```
┌────────────────────────────────────────────────────┐
│                 表示层 (Presentation)              │
│         Web UI  │  API 接口  │  CLI               │
├────────────────────────────────────────────────────┤
│                 业务层 (Business)                  │
│    AgenticSearch  │  DocQA  │  KnowledgeBase     │
├────────────────────────────────────────────────────┤
│                 服务层 (Service)                   │
│      retrieve/  │  scan/  │  llm/  │  agentic/   │
├────────────────────────────────────────────────────┤
│                 数据层 (Data)                      │
│          storage/  │  schema/  │  utils/         │
└────────────────────────────────────────────────────┘
```

**每层职责**：

| 层级 | 职责 | 包含模块 |
|------|------|----------|
| **表示层** | 处理用户交互和展示 | Web 前端、API 接口、CLI |
| **业务层** | 核心业务逻辑处理 | 搜索、问答、知识管理 |
| **服务层** | 提供基础服务能力 | 检索、扫描、LLM 调用 |
| **数据层** | 数据存储和管理 | DuckDB、数据模型 |

### 3.2 事件驱动模式

搜索流程采用事件驱动：

```
用户查询
    │
    ▼
[查询意图分析] ───► 判断是否为闲聊
    │
    ├── 是 ──► 返回闲聊回复
    │
    └── 否 ──► [关键词提取]
                    │
                    ▼
              [文本检索] ◄── rga/ripgrep
                    │
                    ▼
              [知识库查询]
                    │
                    ▼
              [证据处理]
                    │
                    ▼
              [LLM 总结]
                    │
                    ▼
              返回结果
```

### 3.3 代理模式 (Agent Pattern)

ReAct Agent 使用代理模式：

```
┌─────────────────────────────────────────────┐
│           ReAct Agent 工作流程              │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────┐    思考     ┌─────┐    行动       │
│  │Thought│ ──────► │Action│ ──────►       │
│  └─────┘            └─────┘               │
│       │                  │                 │
│       │                  ▼                 │
│       │            ┌─────────┐             │
│       │            │工具执行 │             │
│       │            └────┬────┘             │
│       │                 │                  │
│       ▼                 ▼                  │
│  ┌─────────────────────────────────┐      │
│  │      观察结果 (Observation)      │◄─────┘
│  └─────────────────────────────────┘
│                 │                           │
│                 ▼                           │
│         [判断是否完成]                       │
│           │        │                        │
│          完成    继续                       │
│           │        │                        │
│           └────┬──┘                        │
│                ▼                            │
│         返回最终结果                         │
└─────────────────────────────────────────────┘
```

## 四、数据流向

### 4.1 搜索数据流

```
1. 用户输入查询
   │
   ▼
2. FastAPI 接收请求 (api/search.py)
   │
   ▼
3. AgenticSearch.search() 处理
   │
   ├──► 3.1 查询预处理 (关键词提取)
   │
   ├──► 3.2 调用 GrepRetriever 检索文本
   │         │
   │         ▼
   │    使用 ripgrep-all 搜索文件
   │
   ├──► 3.3 调用 KnowledgeBase 获取知识
   │         │
   │         ▼
   │    从 DuckDB 查询知识集群
   │
   ├──► 3.4 证据处理 (Evidence Processor)
   │         │
   │         ▼
   │    排序、过滤、去重
   │
   └──► 3.5 调用 LLM 生成总结
             │
             ▼
        返回搜索结果
```

### 4.2 知识管理数据流

```
1. 添加文档/知识
   │
   ▼
2. FileScanner / DirScanner 扫描文件
   │
   ▼
3. TextRetriever 提取文本内容
   │
   ▼
4. KnowledgeBase 创建 EvidenceUnit
   │
   ▼
5. DuckDBStorage 保存到数据库
   │
   ▼
6. 索引更新完成，可搜索
```

## 六、扩展点设计

### 6.1 插件化设计

Sirchmunk 支持多种扩展方式：

#### MCP 扩展
- 位置：`src/sirchmunk_mcp/`
- 用途：供 AI Agent 调用

#### 自定义工具
- 位置：`src/sirchmunk/agentic/tools.py`
- 用途：扩展 Agent 能力

#### 扫描器扩展
- 位置：`src/sirchmunk/scan/`
- 支持：文件扫描、目录扫描、网页扫描

### 6.2 可插拔的检索层

```
┌─────────────────────────────────────────┐
│          检索层抽象接口                  │
├─────────────────────────────────────────┤
│  BaseRetriever (retrieve/base.py)       │
│         │                               │
│         ├──► GrepRetriever (默认)       │
│         │        (使用 ripgrep-all)     │
│         │                               │
│         └──► [可扩展其他检索器]          │
└─────────────────────────────────────────┘
```
