# 运行逻辑

> 本笔记详细解析 aimirror 的请求处理流程和状态管理，帮助零基础用户理解程序执行路径。

---

## 一、运行逻辑概述

### 第一步：点出核心流程概念

理解 aimirror 的运行逻辑需要认识5个核心概念：

- **服务启动**：像"开店准备"，加载配置、初始化组件、监听端口
- **请求路由**：像"前台接待"，接收请求并根据规则分发
- **缓存检查**：像"仓库查询"，看文件是否已有存货
- **并发下载**：像"多线程配送"，多个线程同时下载文件分片
- **响应返回**：像"交货"，把文件数据返回给客户端

## 三、关键状态管理

### 1. 全局状态变量

```python
# main.py 中的全局变量
config: dict                         # 配置字典
router: Router                       # 路由器实例
cache: CacheManager                  # 缓存管理器实例
http_client: httpx.AsyncClient       # HTTP客户端实例
download_semaphore: asyncio.Semaphore # 全局并发控制信号量
active_downloads: dict               # 正在下载的任务 {cache_key: (temp_file, task, start_time)}
upstream_proxy: Optional[str]        # 上游代理地址
```

**状态说明**：
- `config`：服务启动时加载，运行期间只读
- `router`：服务启动时初始化，运行期间只读
- `cache`：服务启动时初始化，运行期间读写
- `http_client`：服务启动时创建，运行期间复用
- `download_semaphore`：控制全局并发下载数量
- `active_downloads`：记录正在下载的文件，防止重复下载

### 2. 下载状态转换

```
[新请求]
    │
    ▼
[检查 active_downloads]
    │
    ├─── 无记录: 开始新下载
    │    │
    │    ├─ 创建临时文件
    │    ├─ 创建下载任务
    │    ├─ 记录到 active_downloads
    │    └─ 返回流式响应
    │
    └─── 有记录: 等待正在下载的任务完成
         │
         ├─ 任务完成: 返回缓存文件
         └─ 超时: 返回302重试
```

## 四、并发控制机制

### 1. 两级并发控制

```
全局并发控制 (download_semaphore)
    │
    │  控制同时进行的下载任务总数
    │  默认: 100
    │
    └─── [任务1] ───────┐
         [任务2] ───────┤
         [任务3] ───────┤─── 最多100个同时进行
         ...            │
         [任务100] ─────┘

单任务并发控制 (Semaphore(concurrency))
    │
    │  控制单个任务内的并发分片数
    │  默认: 20
    │
    └─── [分片1] ────┐
         [分片2] ────┤
         [分片3] ────┼─── 单个任务最多20个并发分片
         ...         │
         [分片20] ───┘
```

### 2. 信号量获取流程

```python
# main.py 中的信号量获取
async with download_semaphore:
    # 获取信号量，开始下载
    ...
    # 释放信号量，允许其他任务开始
```

**说明**：
- 使用 `async with` 确保信号量正确释放
- 即使下载失败，信号量也会正确释放
- 高负载时，任务会在信号量外排队等待

### 3. 长轮询机制

**为什么需要长轮询？**
- 避免客户端频繁重试
- 减轻服务器压力
- 提高用户体验

**长轮询实现**：
```python
# 客户端等待正在下载的任务
for _ in range(40):  # 最多等待20秒
    await asyncio.sleep(0.5)  # 每0.5秒检查一次
    
    if download_task.done():  # 检查任务是否完成
        return cached_file  # 完成：返回文件
    
# 超时：返回302让客户端重试
return Response(status_code=302, headers={"Location": url})
```

## 五、错误处理流程

### 1. 网络错误处理

```
[下载分片失败]
    │
    ▼
[重试机制] (最多3次)
    │
    ├─── 重试1: 等待1秒后重试
    ├─── 重试2: 等待2秒后重试
    └─── 重试3: 等待4秒后重试
         │
         ▼
    [全部失败: 抛出 RuntimeError]
         │
         ▼
    [清理临时文件]
         │
         ▼
    [返回错误给客户端]
```

### 2. 超时处理

```
[请求超时] (默认300秒)
    │
    ▼
[连接池超时] (默认5秒)
    │
    ▼
[长轮询超时] (默认20秒)
    │
    ▼
[返回302重试或错误响应]
```

### 3. 客户端断开处理

```python
# main.py 中的客户端断开处理
except asyncio.CancelledError:
    # 客户端断开连接，后台下载继续运行
    logging.info(f"Client disconnected, download continuing in background")
    raise  # 重新抛出，让 FastAPI 处理
```

**说明**：
- 客户端断开连接时，后台下载任务继续运行
- 下载完成后存入缓存供下次使用
- 避免资源浪费
