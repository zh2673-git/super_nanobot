## 概述

本文深入分析 LangExtract 的核心源码，解析关键类、函数和算法的实现细节。

## 第一步：点出核心代码组件概念

LangExtract 源码由以下核心组件构成：

- **[extract 函数](#extract-函数)**：主入口函数，编排整个提取流程，就像"总导演"协调整场演出
- **[RegexTokenizer 类](#regextokenizer-类)**：文本分词器，将长文档切分成小块，就像"裁缝"剪裁布料
- **[BaseLanguageModel 抽象类](#baselanguagemodel-抽象类)**：LLM 提供者的基类，定义通用接口，就像"万能适配器"的规格说明书
- **[PromptBuilder 类](#promptbuilder-类)**：提示词构建器，生成 LLM 指令，就像"编剧"写剧本
- **[AnnotationMerge 类](#annotationmerge-类)**：结果合并器，处理去重和重叠，就像"组装工"拼接零件

```
[extraction.py: extract() 函数]
    |
    +-> [tokenizer.py: RegexTokenizer.tokenize()]
    |       |-> 文本切分
    |       |-> 返回 List[TextChunk]
    |   
    +-> [prompting.py: PromptBuilder.build()]
    |       |-> 构建提示词
    |       |-> 注入示例
    |       |-> 返回 Prompt
    |   
    +-> [inference.py: LanguageModel.generate()]
    |       |-> [providers/gemini.py: GeminiProvider]
    |       |-> [providers/openai.py: OpenAIProvider]
    |       |-> [providers/ollama.py: OllamaProvider]
    |       |-> API 调用和响应解析
    |       |-> 返回 ExtractionResult
    |   
    +-> [schema.py: SchemaValidator.validate()]
    |       |-> 验证提取结果
    |       |-> 返回 ValidatedExtraction
    |   
    +-> [annotation.py: merge()]
        |-> 结果合并
        |-> 去重和重叠处理
        |-> 位置映射
        |-> 返回 AnnotatedDocument
```

**代码关系说明**：extraction.py 的 extract 函数就像"总导演"，它会依次调用 tokenizer（裁缝）切分文本、prompting（编剧）构建提示词、inference（外交官）调用 API、schema（质检员）验证结果、annotation（组装工）合并结果，最终把完整的"演出"（结构化结果）交给观众。

## 第三步：核心代码详细解析

### extract 函数

**专业定义**：`extract` 函数是 LangExtract 的主入口函数，负责接收用户输入、协调各模块、返回结构化提取结果。

**通俗解释与类比**：extract 就像电影拍摄的"总导演"。一部电影需要编剧写剧本、摄影师拍摄、剪辑师剪辑、后期做特效，总导演负责协调所有环节，最终把电影完成。extract 函数同样协调 tokenizer、prompting、inference、schema、annotation 等模块，完成信息提取。

**源码位置**：`langextract/extraction.py`

**核心代码结构**：

```python
def extract(
    text_or_documents,      # 要处理的文本
    prompt_description,    # 提取规则描述
    examples=None,         # few-shot 示例
    model_id="gemini-2.5-flash",  # 使用的模型
    max_char_buffer=1000,   # 每块最大字符数
    extraction_passes=1,   # 提取轮次
    max_workers=10,        # 并行工作数
    # ... 其他参数
):
    # 1. 参数验证和预处理
    # 2. 创建 tokenizer
    tokenizer = tokenizer_lib.RegexTokenizer()
    
    # 3. 文本分块
    chunks = tokenizer.tokenize(document, max_char_buffer)
    
    # 4. 循环处理（支持多轮提取）
    for pass_i in range(extraction_passes):
        # 5. 构建提示词
        prompts = prompting.build_prompts(chunks, prompt_description, examples)
        
        # 6. 并行推理
        results = inference.batch_generate(prompts, model_config, max_workers)
        
        # 7. 验证结果
        validated = schema.validate(results)
        
        # 8. 合并结果
        annotated_doc = annotation.merge(validated, chunks)
    
    return annotated_doc
```

**关键参数说明**：
- `text_or_documents`：输入文本、URL 或 Document 列表
- `prompt_description`：告诉 LLM 要提取什么
- `examples`：few-shot 示例，帮助 LLM 理解任务
- `model_id`：使用的模型（如 "gemini-2.5-flash"）
- `max_char_buffer`：控制每块大小，影响 API 调用次数
- `extraction_passes`：多轮提取可提高召回率
- `max_workers`：并行处理数量，提高速度

### RegexTokenizer 类

**专业定义**：`RegexTokenizer` 是 LangExtract 的默认分词器，使用正则表达式将文本按句子或段落切分。

**通俗解释与类比**：RegexTokenizer 就像一个"智能裁缝"。一块大布料（长文本）没法直接塞进缝纫机（LLM 的上下文窗口），裁缝需要根据布料的特点（句子边界、段落边界）把它剪成合适的小块，同时确保每小块裁剪后仍然有意义。

**源码位置**：`langextract/core/tokenizer.py`

**核心代码结构**：

```python
class RegexTokenizer:
    """使用正则表达式进行文本分词。"""
    
    def __init__(self, pattern: str = None):
        # 默认按句子边界切分
        self.pattern = pattern or r"(?<=[.!?])\s+"
    
    def tokenize(
        self,
        document: "Document",
        max_char_buffer: int = 1000,
        overlap: int = 50
    ) -> list["TextChunk"]:
        """将文档切分成小块。
        
        Args:
            document: 要切分的文档
            max_char_buffer: 每块最大字符数
            overlap: 相邻块之间的重叠字符数
            
        Returns:
            TextChunk 列表
        """
        chunks = []
        text = document.text
        
        # 1. 按句子边界初步切分
        sentences = self._split_sentences(text)
        
        # 2. 将句子组合成块，控制每块大小
        current_chunk = []
        current_length = 0
        
        for sentence in sentences:
            if current_length + len(sentence) > max_char_buffer:
                # 当前块已满，保存并开始新块
                if current_chunk:
                    chunks.append(self._create_chunk(current_chunk, text))
                    # 保持重叠
                    current_chunk = current_chunk[-overlap:]
                    current_length = sum(len(s) for s in current_chunk)
            
            current_chunk.append(sentence)
            current_length += len(sentence)
        
        # 处理最后一个块
        if current_chunk:
            chunks.append(self._create_chunk(current_chunk, text))
        
        return chunks
    
    def _create_chunk(self, sentences: list[str], full_text: str) -> "TextChunk":
        """创建 TextChunk 对象。"""
        chunk_text = " ".join(sentences)
        
        # 计算在原文中的位置
        start = full_text.find(chunk_text)
        end = start + len(chunk_text)
        
        return TextChunk(
            text=chunk_text,
            start=start,
            end=end
        )
```

**关键设计**：
- **句子边界切分**：使用正则表达式 `(?<=[.!?])\s+` 识别句子结束
- **重叠机制**：相邻块之间有重叠，避免边界信息丢失
- **位置记录**：记录每个块在原文中的起止位置

### BaseLanguageModel 抽象类

**专业定义**：`BaseLanguageModel` 是所有 LLM 提供者的抽象基类，定义了统一的接口规范。

**通俗解释与类比**：BaseLanguageModel 就像"万能电源适配器"的规格说明书。无论是什么品牌的电器（Gemini、OpenAI、Ollama），只要按照这个规格说明书生产，就能用统一的接口供电。

**源码位置**：`langextract/core/base_model.py`

**核心代码结构**：

```python
from abc import ABC, abstractmethod

class BaseLanguageModel(ABC):
    """LLM 提供者的抽象基类。"""
    
    @abstractmethod
    def generate(
        self,
        prompt: "Prompt",
        schema: "Schema | None" = None,
        temperature: float | None = None,
        **kwargs
    ) -> "GenerationResult":
        """生成提取结果。
        
        Args:
            prompt: 提示词对象
            schema: 输出模式约束（可选）
            temperature: 采样温度
            
        Returns:
            GenerationResult 对象
        """
        pass
    
    @abstractmethod
    def batch_generate(
        self,
        prompts: list["Prompt"],
        max_workers: int = 10,
        **kwargs
    ) -> list["GenerationResult"]:
        """批量生成提取结果。
        
        Args:
            prompts: 提示词列表
            max_workers: 最大并行数
            
        Returns:
            GenerationResult 列表
        """
        pass
```

**子类实现**：
- **GeminiProvider**：Google Gemini
- **OpenAIProvider**：OpenAI GPT 系列
- **OllamaProvider**：Ollama 本地模型

### PromptBuilder 类

**专业定义**：`PromptBuilder` 负责构建发送给 LLM 的完整提示词，包括任务描述、few-shot 示例和输出格式要求。

**通俗解释与类比**：PromptBuilder 就像"编剧"写剧本。编剧需要考虑：这是什么场景（任务类型）、演员要说什么台词（提取规则）、要不要给演员做示范（few-shot 示例）、最终要什么形式的表演（输出格式）。

**源码位置**：`langextract/prompting.py`

**核心代码结构**：

```python
class PromptBuilder:
    """提示词构建器。"""
    
    def __init__(
        self,
        prompt_description: str,
        examples: list["ExampleData"],
        format_type: str = "json"
    ):
        self.prompt_description = prompt_description
        self.examples = examples
        self.format_type = format_type
    
    def build(self, chunk: "TextChunk") -> "Prompt":
        """为文本块构建提示词。"""
        
        # 1. 构建任务描述部分
        task_description = self._build_task_description()
        
        # 2. 构建 few-shot 示例
        examples_section = self._build_examples()
        
        # 3. 构建输出格式要求
        format_section = self._build_format()
        
        # 4. 组合成完整提示词
        full_prompt = f"""{task_description}

{examples_section}

{format_section}

Input text:
{chunk.text}
"""
        
        return Prompt(
            text=full_prompt,
            chunk=chunk
        )
    
    def _build_examples(self) -> str:
        """构建 few-shot 示例部分。"""
        if not self.examples:
            return ""
        
        examples_text = "Example extractions:\n"
        for i, example in enumerate(self.examples, 1):
            examples_text += f"\nExample {i}:\n"
            examples_text += f"Input: {example.text}\n"
            examples_text += "Extractions:\n"
            for ext in example.extractions:
                examples_text += f"""  - class: {ext.extraction_class}
    text: {ext.extraction_text}
    attributes: {ext.attributes}
"""
        return examples_text
```

### AnnotationMerge 类

**专业定义**：`AnnotationMerge` 负责合并多个文本块的提取结果，处理去重、重叠和位置映射。

**通俗解释与类比**：AnnotationMerge 就像"组装工"拼接零件。裁缝把布料剪成小块，加工后再由组装工把成品零件拼接成完整的产品。组装工需要识别重复的零件、处理重叠的边界、确保每个零件都在正确的位置。

**源码位置**：`langextract/annotation.py`

**核心代码结构**：

```python
class AnnotationMerge:
    """提取结果合并器。"""
    
    def merge(
        self,
        extractions: list["ValidatedExtraction"],
        chunks: list["TextChunk"]
    ) -> "AnnotatedDocument":
        """合并提取结果。"""
        
        # 1. 去重
        unique_extractions = self._deduplicate(extractions)
        
        # 2. 处理重叠
        merged_extractions = self._resolve_overlaps(unique_extractions)
        
        # 3. 位置映射
        mapped_extractions = self._map_positions(merged_extractions, chunks)
        
        return AnnotatedDocument(
            text=chunks[0].text,  # 保留原文
            extractions=mapped_extractions
        )
    
    def _deduplicate(self, extractions: list) -> list:
        """去除重复的提取结果。"""
        seen = set()
        unique = []
        
        for ext in extractions:
            # 用提取文本和类别作为唯一键
            key = (ext.extraction_text, ext.extraction_class)
            
            if key not in seen:
                seen.add(key)
                unique.append(ext)
        
        return unique
    
    def _resolve_overlaps(self, extractions: list) -> list:
        """处理重叠的提取。"""
        # 按位置排序
        sorted_exts = sorted(extractions, key=lambda x: x.source_start)
        
        merged = []
        current = None
        
        for ext in sorted_exts:
            if current is None:
                current = ext
                continue
            
            # 如果有重叠
            if ext.source_start < current.source_end:
                # 选择更早出现的或更长的
                if len(ext.extraction_text) > len(current.extraction_text):
                    current = ext
            else:
                merged.append(current)
                current = ext
        
        if current:
            merged.append(current)
        
        return merged
    
    def _map_positions(
        self,
        extractions: list,
        chunks: list["TextChunk"]
    ) -> list:
        """将块内位置映射到原文位置。"""
        mapped = []
        
        for ext in extractions:
            chunk = ext.chunk
            
            # 块内位置 + 块起始位置 = 原文位置
            source_start = chunk.start + ext.chunk_start
            source_end = chunk.start + ext.chunk_end
            
            mapped_ext = ExtractedAnnotation(
                extraction_class=ext.extraction_class,
                extraction_text=ext.extraction_text,
                attributes=ext.attributes,
                source_start=source_start,
                source_end=source_end
            )
            mapped.append(mapped_ext)
        
        return mapped
```
