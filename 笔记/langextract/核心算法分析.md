# LangExtract 核心算法分析

## 概述

LangExtract 的核心算法主要涉及文本分块、LLM输出解析和文本对齐三个关键环节。其中最重要的算法是 **WordAligner 文本对齐算法**，它负责将 LLM 提取的结果精确映射到源文本中的位置。这个算法解决了"大海捞针"问题——如何在一个长文本中找到 LLM 提取的那个精确片段。

---

## 第一步：点出核心算法组件概念

LangExtract 包含以下核心算法组件：

- **[WordAligner 类](#wordaligner-类)**：文本对齐器，就像"定位仪"或"GPS导航仪"，负责将提取结果精确映射到源文本中的位置
- **[精确匹配算法](#精确匹配算法)**：使用 difflib SequenceMatcher 进行精确匹配，就像"精确制导导弹"，能精准命中目标
- **[模糊匹配算法](#模糊匹配算法)**：使用滑动窗口+相似度计算进行模糊匹配，就像"雷达搜索"，在精确匹配失败时兜底查找
- **[文本分块算法](#文本分块算法)**：将长文档切分成小块的策略，就像"裁缝剪裁布料"，把大布料剪成小布料
- **[结果合并算法](#结果合并算法)**：处理多个块的提取结果去重和重叠，就像"组装工"拼接零件

## 第二步：用图构建算法流程

### WordAligner 整体算法流程

```
[LLM 输出：提取结果列表]
    |
    v
[第一步：文本分词]
    |-> 将源文本分词：source_tokens
    |-> 将提取结果分词：extraction_tokens
    |
    v
[第二步：精确匹配阶段]
    |-> 使用 difflib.SequenceMatcher 找完全匹配
    |-> 匹配成功 -> 设置 token_interval 和 char_interval
    |-> 状态标记：MATCH_EXACT（精确匹配）或 MATCH_LESSER（部分匹配）
    |
    v
[第三步：收集未匹配项]
    |-> 将未能精确匹配的提取结果收集到 unaligned_extractions
    |
    v
[第四步：模糊匹配阶段]
    |-> 对每个未匹配的提取结果
    |   |-> 使用滑动窗口遍历源文本的所有可能位置
    |   |-> 对每个窗口计算与提取结果的相似度
    |   |-> 相似度 >= 阈值 -> 设置位置信息
    |   |-> 状态标记：MATCH_FUZZY（模糊匹配）
    |
    v
[第五步：返回结果]
    |-> 返回带有位置信息的提取结果列表
    |-> 每个 extraction 包含：
    |   |-> token_interval：token级别的起止位置
    |   |-> char_interval：字符级别的起止位置
    |   |-> alignment_status：对齐状态
```

**算法流程说明**：整个对齐过程就像警察（WordAligner）追踪嫌疑人的过程。首先，警察获取嫌疑人的照片（提取结果），然后在监控录像（源文本）中精确搜索匹配的人（精确匹配阶段）。如果找不到，就使用模糊搜索，根据身高、衣着等特征在监控中寻找相似的人（模糊匹配阶段）。最终，警察会确定嫌疑人在监控中的精确出现时间段（位置信息）。

### 精确匹配算法

**专业定义**：精确匹配算法使用 Python 的 `difflib.SequenceMatcher` 在源文本中寻找与提取结果完全匹配的文本片段。

**通俗解释与类比**：精确匹配就像在字典中查一个单词。你知道这个单词的准确拼写，直接在字典中查找就能找到它的页码。这个算法的前提是 LLM 提取的文本与源文本几乎完全一致（可能有大小写或轻微差异）。

**算法原理**：

1. **文本分词**：将源文本和提取结果都分成单词（tokens）
2. **构建匹配器**：使用 `difflib.SequenceMatcher` 比较两组 tokens
3. **获取匹配块**：调用 `get_matching_blocks()` 获取所有匹配的区间
4. **设置位置信息**：根据匹配的起始位置和长度，计算 token 级别的位置和字符级别的位置

**核心代码逻辑**：

```python
# 精确匹配阶段
for i, j, n in self._get_matching_blocks()[:-1]:
    extraction, _ = index_to_extraction_group.get(j, (None, None))
    if extraction is None:
        continue
    
    # 设置 token 级别的位置
    extraction.token_interval = tokenizer_lib.TokenInterval(
        start_index=i + token_offset,
        end_index=i + n + token_offset,
    )
    
    # 设置字符级别的位置
    start_token = tokenized_text.tokens[i]
    end_token = tokenized_text.tokens[i + n - 1]
    extraction.char_interval = data.CharInterval(
        start_pos=char_offset + start_token.char_interval.start_pos,
        end_pos=char_offset + end_token.char_interval.end_pos,
    )
    
    # 根据匹配程度设置状态
    if extraction_text_len == n:
        extraction.alignment_status = data.AlignmentStatus.MATCH_EXACT
    else:
        extraction.alignment_status = data.AlignmentStatus.MATCH_LESSER
```

**匹配状态说明**：

| 状态 | 含义 | 场景 |
|------|------|------|
| `MATCH_EXACT` | 精确匹配 | LLM 提取的文本与源文本完全一致 |
| `MATCH_LESSER` | 部分匹配 | 提取的文本比匹配的源文本更长（如包含标点） |
| `MATCH_FUZZY` | 模糊匹配 | 精确匹配失败，使用模糊算法找到最佳匹配 |
| `None` | 未匹配 | 无法找到匹配 |

### 模糊匹配算法

**专业定义**：模糊匹配算法是精确匹配的"兜底方案"，当精确匹配失败时，使用滑动窗口+相似度计算在源文本中寻找与提取结果最相似的文本片段。

**通俗解释与类比**：模糊匹配就像在一个模糊的照片中寻找目标人物。你知道目标的大致特征（身高、衣着），但无法精确匹配。于是你遍历监控录像的每一帧，对每帧计算与目标的相似度，找到最相似的那一帧。模糊匹配同样如此——它遍历源文本的所有可能位置，计算每个位置与提取结果的相似度，找到最相似的那个。

**算法原理**：

1. **预检查优化**：使用 token 计数交集快速排除不可能匹配的窗口
2. **滑动窗口**：遍历源文本中所有长度可能的窗口
3. **相似度计算**：对每个窗口使用 SequenceMatcher 计算相似度
4. **阈值判断**：如果相似度 >= 阈值（默认0.75），则认为匹配成功

**核心代码逻辑**：

```python
def _fuzzy_align_extraction(
    self,
    extraction: data.Extraction,
    source_tokens: list[str],
    tokenized_text: tokenizer_lib.TokenizedText,
    token_offset: int,
    char_offset: int,
    fuzzy_alignment_threshold: float = 0.75,
    tokenizer_impl: tokenizer_lib.Tokenizer | None = None,
) -> data.Extraction | None:
    """使用 difflib.SequenceMatcher 在 token 级别进行模糊对齐。"""
    
    # 1. 将提取结果分词
    extraction_tokens = list(
        _tokenize_with_lowercase(extraction.extraction_text, tokenizer_inst=tokenizer_impl)
    )
    extraction_tokens_norm = [_normalize_token(t) for t in extraction_tokens]
    
    # 2. 预检查：计算提取结果的 token 计数
    extraction_counts = collections.Counter(extraction_tokens_norm)
    min_overlap = int(len(extraction_tokens) * fuzzy_alignment_threshold)
    
    best_ratio = 0.0
    best_span: tuple[int, int] | None = None
    
    # 3. 滑动窗口遍历所有可能位置
    len_e = len(extraction_tokens)
    max_window = len(source_tokens)
    
    for window_size in range(len_e, max_window + 1):
        window_deque = collections.deque(source_tokens[0:window_size])
        window_counts = collections.Counter(
            [_normalize_token(t) for t in window_deque]
        )
        
        for start_idx in range(len(source_tokens) - window_size + 1):
            # 预检查：快速排除不可能匹配的窗口
            if (extraction_counts & window_counts).total() >= min_overlap:
                # 计算相似度
                window_tokens_norm = [_normalize_token(t) for t in window_deque]
                matcher.set_seq1(window_tokens_norm)
                matches = sum(size for _, _, size in matcher.get_matching_blocks())
                ratio = matches / len_e if len_e > 0 else 0.0
                
                if ratio > best_ratio:
                    best_ratio = ratio
                    best_span = (start_idx, window_size)
            
            # 滑动窗口右移
            if start_idx + window_size < len(source_tokens):
                # 移除左边 token
                old_token = window_deque.popleft()
                old_token_norm = _normalize_token(old_token)
                window_counts[old_token_norm] -= 1
                if window_counts[old_token_norm] == 0:
                    del window_counts[old_token_norm]
                
                # 添加新 token
                new_token = source_tokens[start_idx + window_size]
                window_deque.append(new_token)
                new_token_norm = _normalize_token(new_token)
                window_counts[new_token_norm] += 1
    
    # 4. 判断是否找到匹配
    if best_span and best_ratio >= fuzzy_alignment_threshold:
        start_idx, window_size = best_span
        # 设置位置信息
        extraction.token_interval = tokenizer_lib.TokenInterval(
            start_index=start_idx + token_offset,
            end_index=start_idx + window_size + token_offset,
        )
        extraction.alignment_status = data.AlignmentStatus.MATCH_FUZZY
        return extraction
    
    return None
```

**模糊匹配的关键优化**：

1. **Token 计数交集预检查**：在计算昂贵的相似度之前，先用简单的计数判断是否可能匹配
2. **滑动窗口复用**：使用 deque 数据结构，高效地滑动窗口
3. **轻量级词干处理**：使用 `_normalize_token` 函数进行轻量级的词干提取（主要是处理复数），避免 "characters" 和 "character" 无法匹配的问题

**Normalization 示例**：
```python
def _normalize_token(token: str) -> str:
    """小写并应用轻量级词干提取。"""
    token = token.lower()
    if len(token) > 3 and token.endswith("s") and not token.endswith("ss"):
        token = token[:-1]  # 去掉复数后缀
    return token
```

### 文本分块算法

**专业定义**：文本分块算法将长文档切分成适合 LLM 处理的小块，同时保持语义连贯性和边界信息。

**通俗解释与类比**：文本分块就像"裁缝剪裁布料"。一块大布料（长文本）没法直接塞进缝纫机（LLM 的上下文窗口），裁缝需要根据布料的特点（句子边界、段落边界）把它剪成合适的小块，同时确保每小块裁剪后仍然有意义。

**算法策略**：

1. **按句子边界切分**：优先在句子边界（句号、问号、感叹号后）切分
2. **控制块大小**：使用 `max_char_buffer` 参数控制每块的最大字符数
3. **重叠机制**：相邻块之间有重叠（默认50字符），避免边界信息丢失
4. **位置记录**：记录每个块在原文中的起止位置

**核心代码逻辑**：

```python
class RegexTokenizer:
    def tokenize(
        self,
        document: "Document",
        max_char_buffer: int = 1000,
        overlap: int = 50
    ) -> list["TextChunk"]:
        """将文档切分成小块。"""
        
        # 1. 按句子边界初步切分
        sentences = self._split_sentences(text)
        
        chunks = []
        current_chunk = []
        current_length = 0
        
        for sentence in sentences:
            # 2. 检查是否需要开始新块
            if current_length + len(sentence) > max_char_buffer:
                if current_chunk:
                    # 保存当前块
                    chunks.append(self._create_chunk(current_chunk, text))
                    # 3. 保持重叠
                    current_chunk = current_chunk[-overlap:]
                    current_length = sum(len(s) for s in current_chunk)
            
            current_chunk.append(sentence)
            current_length += len(sentence)
        
        # 处理最后一个块
        if current_chunk:
            chunks.append(self._create_chunk(current_chunk, text))
        
        return chunks
```

### 结果合并算法

**专业定义**：结果合并算法负责将多个文本块的提取结果合并成最终的结构化文档，处理去重、重叠和位置映射。

**通俗解释与类比**：结果合并就像"组装工"拼接零件。裁缝把布料剪成小块，加工后再由组装工把成品零件拼接成完整的产品。组装工需要识别重复的零件、处理重叠的边界、确保每个零件都在正确的位置。

**算法步骤**：

1. **去重**：识别跨块的重复提取，保留首次出现的
2. **处理重叠**：处理边界重叠的提取，选择更完整或更早出现的结果
3. **位置映射**：将块内相对位置转换为原文绝对位置

**核心代码逻辑**：

```python
class AnnotationMerge:
    def merge(
        self,
        extractions: list["ValidatedExtraction"],
        chunks: list["TextChunk"]
    ) -> "AnnotatedDocument":
        """合并提取结果。"""
        
        # 1. 去重
        unique_extractions = self._deduplicate(extractions)
        
        # 2. 处理重叠
        merged_extractions = self._resolve_overlaps(unique_extractions)
        
        # 3. 位置映射
        mapped_extractions = self._map_positions(merged_extractions, chunks)
        
        return AnnotatedDocument(
            text=chunks[0].text,
            extractions=mapped_extractions
        )
    
    def _deduplicate(self, extractions: list) -> list:
        """去除重复的提取结果。"""
        seen = set()
        unique = []
        
        for ext in extractions:
            key = (ext.extraction_text, ext.extraction_class)
            if key not in seen:
                seen.add(key)
                unique.append(ext)
        
        return unique
    
    def _resolve_overlaps(self, extractions: list) -> list:
        """处理重叠的提取。"""
        sorted_exts = sorted(extractions, key=lambda x: x.source_start)
        
        merged = []
        current = None
        
        for ext in sorted_exts:
            if current is None:
                current = ext
                continue
            
            # 如果有重叠
            if ext.source_start < current.source_end:
                # 选择更长的
                if len(ext.extraction_text) > len(current.extraction_text):
                    current = ext
            else:
                merged.append(current)
                current = ext
        
        if current:
            merged.append(current)
        
        return merged
    
    def _map_positions(self, extractions: list, chunks: list["TextChunk"]) -> list:
        """将块内位置映射到原文位置。"""
        mapped = []
        
        for ext in extractions:
            chunk = ext.chunk
            # 块内位置 + 块起始位置 = 原文位置
            source_start = chunk.start + ext.chunk_start
            source_end = chunk.start + ext.chunk_end
            
            mapped_ext = ExtractedAnnotation(
                extraction_class=ext.extraction_class,
                extraction_text=ext.extraction_text,
                attributes=ext.attributes,
                source_start=source_start,
                source_end=source_end
            )
            mapped.append(mapped_ext)
        
        return mapped
```

## 算法性能分析

### 时间复杂度

| 算法 | 时间复杂度 | 说明 |
|------|-----------|------|
| 精确匹配 | O(n+m) | SequenceMatcher 的时间复杂度，其中 n 和 m 分别是源文本和提取结果的长度 |
| 模糊匹配 | O(n×m×k) | 最坏情况，其中 n 是源文本长度，m 是提取结果长度，k 是窗口滑动次数 |
| 文本分块 | O(n) | 线性扫描文本并按句子边界切分 |
| 结果合并 | O(n log n) | 排序处理重叠，最坏 O(n log n) |

### 空间复杂度

| 算法 | 空间复杂度 | 说明 |
|------|-----------|------|
| 精确匹配 | O(n+m) | 需要存储源文本和提取结果的分词结果 |
| 模糊匹配 | O(n+m) | 需要存储滑动窗口和计数统计 |
| 文本分块 | O(n) | 需要存储切分后的块列表 |
| 结果合并 | O(n) | 需要存储合并后的结果列表 |

## 算法应用场景

### 1. 临床笔记结构化

在医疗领域，医生需要从大量的临床笔记中提取诊断、药物、症状等信息。WordAligner 算法可以精确地将提取的医学实体映射回原文，方便医生审核和验证。

### 2. 法律文档分析

在法律领域，需要从冗长的判例文书或合同中提取关键条款、日期、金额等信息。模糊匹配算法可以处理 LLM 输出与原文略有差异的情况。

### 3. 金融报告提取

在金融领域，需要从年报、季报中提取财务数据、关键指标等。文本分块算法可以处理超长文档，结果合并算法可以将多个章节的提取结果整合。
